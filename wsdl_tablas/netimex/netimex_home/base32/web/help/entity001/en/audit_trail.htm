Audit Trail

The Bank can configure what type of activities are to be logged and can generate an inquiry on the audited information.

A manual purge facility allows the bank to delete audit information from the database using flexible criteria. There is also a background service for automatic purge based on a retention period.

The following activities can be logged:

  • Transaction Processing
  • User Profile Maintenance
  • User authorization changes
  • Signature Levels Changes
  • Banks and Parties Maintenance
  • Bank General Parameters Maintenance
  • Translation

Audit Trail Inquiry

The Bank can inquire upon the audited information using a flexible search facility for the following criteria: Activity, Customer, User and/or Dates From and To. In order to display a list of all the audited information the user must leave all the criteria fields empty and click on the search button.

Audit Trail List

The user can sort the list per column by clicking on the column heading.

Clear Log Activities

The user may clear the Log Activities for an Activity, Customer and/or User for a specific period.

In order to clear Log Activities:

  • Click on the Delete Icon   on the previous screen.
  • Select the relevant criteria.
  • Click on the Delete button.
