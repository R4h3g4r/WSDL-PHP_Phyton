REMITTANCE

Overview of Remittance

A remittance is defined as a payment made for any purpose including a payment made by an importer to an exporter via a bank.  The payment is in reference to documents, which were not previously handled by the bank.  Since the bank handles no documents, these payments are referred to as "Clean Payments".

Establishing a New Remittance

To establish a new Remittance, click on the Remittance link or the Template icon on the Main Menu.  The Remittance application form appears.  When the Template icon is selected, the Remittance form will appear with certain fields already completed.

The sections that must be completed in order to request a Remittance are listed on the yellow Navigation Toolbar that appears before every section.

  • General Details

  • Participants

  • Text

Clicking on any heading in the Navigation Toolbar from anywhere in the application form will bring you to that section.

General Details

The first part of the Remittance Request application form contains the General Details section. This section appears when you first open a Remittance Request.

To complete the General Details section, enter the relevant fields. Some of these fields are mandatory. They are indicated in Bold on the screen and by this icon  in the User Guide.

The General Details section requests the following information:

    Customer Reference - A unique deal identification number decided by the NetIMEX user, for example, a contract number, a delivery order.

Customer Ref Not Required - When this checkbox is checked it indicates that the reference number will automatically be generated. When not checked it indicates that the reference number will not be generated and the user must enter the reference number manually.

    Amount - The amount of the Remittance Application.

    Currency - The currency in which the Application is expressed.

Beneficiary Account Number - The account number to which the payment is made.

Deal Number - This field contains the Deal Number created by the Bank. Note: When NetIMEX is configured to do so, this field will include the bank deal number as generated by NetIMEX.

    Value Date - The stated value of the remittance is pegged to this date.

    Payment Reason - This specifies the reason for the remittance.

    Payment Method - How the payment will be made. Methods include SWIFT, Telex, banker's draft, cheques, etc.

    Payment Dispatch - How the payment will arrive at the Beneficiaries' Bank.  This field is required for any payment method except SWIFT or Telex.

Participants

The second part of the Remittance Request form contains the Participants section. Scroll down from the General Details to access this section.

You may also access the Participants section from anywhere in the application form. Clicking on the Participants heading on the yellow Navigation Toolbar that precedes each section will bring you to this section.

The following list describes the fields in the Participants section:

Beneficiary and Paying Bank Details - Specifies the Beneficiary and Paying Bank name and location details relevant to the Remittance.  Enter the Beneficiary or Paying Bank details in the respective blocks. The user may select a Beneficiary or Paying Bank by entering a code number or by filling in the name and location details. If the code is entered in the database, the name and location details appear automatically when the code is keyed in.  If the required Beneficiary or paying Bank code does not exist in the database, leave the code field blank and type the full name and address of the Beneficiary or paying Bank in the fields provided.

Search by Code - The database codes that identify the Beneficiary or Paying Bank.

Note: The user can identify a specific Party or Bank by entering any number of characters in the Search by Code field and then moving to another field e.g. Enter B and the details of the first Party/Bank that starts with B will be displayed.

Search by Name - The database names of the Beneficiary or Paying Bank.

Name - The Beneficiary or Paying Bank branch name.  There may be several branches defined for a beneficiary. To define participant details for a new Remittance, enter the full Beneficiary or paying Bank name.

Street - The Street name where the Beneficiary or Paying Bank is located.

City - The City where the Beneficiary or Paying Bank is located.

Country - The Country where the Beneficiary or Paying Bank is located.

Default Bank: When a specific Beneficiary is entered in the Participants section, the Paying Bank of the last deal using that Beneficiary together with the currency of that last deal will automatically appear in the Paying Bank fields as the Default value.

Note: The Beneficiary details are always mandatory.  If the Paying Bank details are not entered, the Paying Bank will be selected by the Issuing Bank.

Additional Parties

The number of Additional Parties sections may differ depending on configuration. Each section enables the user to enter information about an additional Party associated with the transaction.

To define a Party:

1.    Select the Party Type.

2.    The system will display the relevant Codes for the select Party Type.

The following fields appear in each Additional Party section:

Additional Party Details - Enter the Party details in the respective blocks. The user may select a Party by entering a code number or by filling in the name and location details. If the code is entered in the database, the name and location details appear automatically when the code is keyed in.  If the required Party code does not exist in the database, leave the code field blank and type the full name and address of the Party in the fields provided.          

Search by Code - The database codes that identify the Party. The user can identify a specific Party by entering any number of characters in the Search by Code field and then moving to another field e.g. Enter B and the details of the first Party that starts with B will be displayed.

Search by Name - The database names of the Party.

Name - The Party branch name.  There may be several branches defined for a beneficiary. To define participant details for a new transaction, enter the full Party name.

Street - The Street name where the Party is located.

City - The City where the Party is located.

Country - The Country where the Party is located.

Texts

The third part of the Remittance form contains the Texts section. Scroll down from the Participants section to access this section. This window enables you to enter additional information relevant to the Remittance and enter account details. Additional information can be entered for these categories and may include other custom‑defined categories.

You may also access the Texts section from anywhere in the application form. Clicking on the Text heading on the yellow Navigation Toolbar that precedes each section will bring you to this section.

This section enables you to enter additional information relevant to the Remittance and enter account details.  Additional information can be entered for the following categories:

Special Instructions - Used to enter additional customer instructions. To enter other deal information, click the appropriate option in the toolbar, or click the directional button to view a preceding window.

Payment Details - Used to enter additional Payment details.

Zoom Text Icon  – Click on this icon to enlarge the text box area. You may edit or enter new data in the enlarged text box.

Open Clause Library Icon  - When clicking on this icon a list of defined clauses relevant for the window are displayed. After selecting the required clauses the contents of the clause is copied to the window.

Clause Library
In order to display the clauses for a specific text window click on the Open Clause Library Icon next to the relevant window.

On the Clause Library window select the clause/s that should be entered into the text by checking the relevant checkbox and clicking on the Select button. In order to include all the Clauses listed in the Clause Library check the checkbox at the top of the column and click on the Select button.

When returning to the Text Screen the user can view the detailed description of the Clause which will be included in the Text. In order to remove a clause from the Text the user must delete the text of the clause from the relevant window.

Defined Clause Descriptions
An additional option exists to display clauses as defined in the clauses form, which the user can select by selecting the desired checkboxes. This replaces the use of the Clause Library and free text windows.

There are two options for this function, which is established during bank setup:

  • The user can select more than one clause from one text window.

  • The user can select only one clause from one text window.

In a text window which is marked as mandatory, a validity check is performed to ensure that at least one clause is selected.

Loan Request

To request the bank to finance the transaction, you may apply to the bank for a loan.

The following list describes the fields in the Loan Request Form:

Amount - Specifies the loan amount.

Currency - Specifies the loan currency.

Loan Type - Specifies the type of loan being requested.

Value Date - Specifies the date the loan becomes effective.

Period - The length of the loan, in days.

Maturity - Specifies the repayment date of the loan if the period is not defined.

Making a Loan Request

To make one or several loan requests:

1.       Enter loan request amount.

2.       Enter the currency code for the loan.

3.       Select the loan type from the drop‑down menu.

4.       Enter the value date.

5.       Either enter the loan period (in days), or the date of maturity.

6.       Use the free format text window to add additional loan request details.

Account Details

The Customer Account Details Window enables you to instruct the bank how to pay the Remittance. 

Up to three Bank Charges can be defined. If data exists in the Payment Item field a corresponding Account number must be entered. If there is a blank Payment Item field, when relevant, any additional Payment Item and corresponding Account number may be selected.

To specify the customer’s account details, select the appropriate field on the Account Instructions Form. The Bank Charges describes the payment item to be paid, for example, Principal Amount, Charges. The Account Number selected from the drop-down list is from the Customer Profile. Use the Additional Instructions window to add additional payment instructions.

Additional Information

This section includes additional details of the transaction. If a field header is bold it indicates that the field is mandatory and the user must enter the relevant text in the corresponding data entry field. If the field header is not bold it is not mandatory and the user does not have to enter data.
