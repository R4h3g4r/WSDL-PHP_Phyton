SHIPPING GUARANTEE

Overview of Shipping Guarantees

A shipping guarantee is a promise from the bank to the shipping company to indemnify the shipping company in case of a claim against it. 

Shipping guarantees allow a shipping company, airline or carrier to release goods to an importer before the title documents have actually been received.  Goods often arrive in the importer’s country before the documents, which allow the importer to take control of the goods.  A shipping company may only release goods upon presentation of the relevant documents.  If the goods are released before receipt of these documents, the shipping company could face legal action from any interested party. Shipping guarantees have no expiry date.

To Establish a Shipping Guarantee

To request a Shipping Guarantee, click on the Shipping Guarantee option from the Import section of the Main Menu. 

The Shipping Guarantee application is displayed. A New request can be initiated from this page. New requests can be updated until they have been sent to the Bank.

The sections that must be completed in order to request a new Shipping Guarantee are listed on the yellow Navigation Toolbar that appears before every section. Clicking on any heading in the Navigation Toolbar from anywhere in the application form will bring you to that section.

  • General Details

  • Participants

  • Text

Once all the issue information is entered, saving your work ensures the deal is updated in the NETIMEX database.  Before the deal is added to the Release List, the deal must be signed.  To sign the completed deal, click the Sign icon and enter your password.

General Details

The first part of the Shipping Guarantee Request application form contains the General Details section. This section appears when you first open a Shipping Guarantee Request.

To complete the General Details section, enter the relevant fields. Some of these fields are mandatory. They are indicated in Bold on the screen and by this icon  in this User Guide.

The General Details section requests the following information:

Transport Document No - The shipping document number.

Ref No Not Required – When this checkbox is checked it indicates that the transport document number will automatically be generated. When not checked indicates that the transport document number will not be generated and the user must enter the transport document number manually.

    Amount - The amount of the Shipping Guarantee request.

    Currency - The currency in which the request is expressed.

Deal Number - This field contains the Deal Number created by the Bank.      
Note: When NetIMEX is configured to do so, this field will include the bank deal number as generated by NetIMEX.

Transportation Name - The name of the transport vessel or transportation company (i.e. name of ship or airline company).

Storage Amount - The value of the goods to be placed in storage (if relevant).

Warehouse - This is the name of the warehouse where the goods will be stored.  This field must be filled in if the Storage Amount is given.

L/C Number - Each L/C is assigned a unique identifying number by the Issuing Bank.

    Send Guarantee by - You can choose how to send the request to the shipping company from this drop-down list, for example, by airmail, courier, mail.

Shipment By Sea - Check this box if the goods arrived by sea transport.

Documents to Order of Bank - Check this box if the shipping documents are in the bank's name.

On Shore/Off Shore/Irrelevant - Select a radio button to indicate if it is an On Shore Deal or Off Shore Deal or Irrelevant. An offshore deal is one where the goods are shipped between two foreign countries.

Participants

The second part of the Shipping Guarantee Request form contains the Participants section. Scroll down from the General Details to access this section.

You may also access the Participants section from anywhere in the application form. Clicking on the Participant title on any heading of the yellow Navigation Toolbar that precedes each section will bring you to this section.

This portion of the application contains the information that defines the Shipping Company to whom the request is being issued.

Note: The user can identify a specific Party or Bank by entering any number of characters in the Search by Code field and then moving to another field e.g. Enter B and the details of the first Party/Bank that starts with B will be displayed.

Search by Code - This is a code name of the Shipping Company Branch who is shipping the goods. By default NetIMEX always displays the Name and Head Office location of the selected Shipping Company.  If the required Shipping Company does not exist, ignore this field and type the full name and address of the Shipping Company in the fields provided.

Search by Name – The user may search for the name of the Shipping Company Branch who is shipping the goods. By default NetIMEX always displays the Code and Head Office location of the selected Shipping Company.

Name, Street, City, Country - This is the name and address of the Shipping Company.  These fields must be completed if the Shipping Company is not defined in NetIMEX.

Additional Parties

The number of Additional Parties sections may differ depending on configuration. Each section enables the user to enter information about an additional Party associated with the transaction.

To define a Party:

1.       Select the Party Type.

2.       The system will display the relevant Codes for the select Party Type

Enter the Additional Party details in the respective blocks. The user may select a Party by entering a code number or by filling in the name and location details. If the code is entered in the database, the name and location details appear automatically when the code is keyed in.  If the required Party code does not exist in the database, leave the code field blank and type the full name and address of the Party in the fields provided.

The following fields appear in each Additional Party section:

Search by Code - The database codes that identify the Party. The user can identify a specific Party by entering any number of characters in the Search by Code field and then moving to another field e.g. Enter B and the details of the first Party that starts with B will be displayed.

Search by Name - The database names of the Party.

Name - The Party branch name.  There may be several branches defined for a beneficiary. To define participant details for a new transaction, enter the full Party name.

Street - The Street name where the Party is located.

City - The City where the Party is located.

Country - The Country where the Party is located.

Texts

The third part of the Shipping Guarantee Request form contains the Texts section. Scroll down from the Participants to access this section.

You may also access the Texts section from anywhere in the application form. Clicking on the Text heading of the yellow Navigation Toolbar that precedes each section will bring you to this section.

This window enables you to enter additional information relevant to the Shipment Guarantee and enter account details.  Additional information can be entered for the following categories:

Comments - Used enter additional comments. Enter additional instructions in the free format text window (Comments field) to specify account details according to the payment item

Customer Instructions - Used to enter additional customer instructions.

Zoom Text Icon  - Click on this icon to enlarge the text box area. You may edit or enter new data in the enlarged text box.

Open Clause Library Icon  - When clicking on this icon a list of defined clauses relevant for the window are displayed. After selecting the required clauses the contents of the clause is copied to the window.

Clause Library

In order to display the clauses for a specific text window click on the Open Clause Library Icon next to the relevant window.

On the Clause Library window select the clause/s that should be entered into the text by checking the relevant checkbox and clicking on the Select button. In order to include all the Clauses listed in the Clause Library check the checkbox at the top of the column and click on the Select button.

When returning to the Text Screen the user can view the detailed description of the Clause which will be included in the Text. In order to remove a clause from the Text the user must delete the text of the clause from the relevant window.

Defined Clause Descriptions
An additional option exists to display clauses as defined in the clauses form, which the user can select by selecting the desired checkboxes. This replaces the use of the Clause Library and free text windows.

There are two options for this function, which is established during bank setup:

  • The user can select more than one clause from one text window.

  • The user can select only one clause from one text window.

In a text window which is marked as mandatory, a validity check is performed to ensure that at least one clause is selected.

Account Details

The Account Details Window enables you to instruct the bank how to make payment for the Shipping Guarantee. 

Up to three Bank Charges can be defined. If data exists in the Payment Item field a corresponding Account number must be entered. If there is a blank Payment Item field, when relevant, any additional Payment Item and corresponding Account number may be selected.

To specify the customer’s account details, select the appropriate field on the Account Instructions Form. The Bank Charges describes the payment item to be paid, for example, Principal Amount, Charges. The Account Number selected from the drop-down list is from the Customer Profile. Use the Additional Instructions window to add additional payment instructions.

Additional Information

This section includes additional details of the transaction. If a field header is bold it indicates that the field is mandatory and the user must enter the relevant text in the corresponding data entry field. If the field header is not bold it is not mandatory and the user does not have to enter data.
