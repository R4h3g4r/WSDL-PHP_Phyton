Entity Details

Entity ID

Clicking on the Entity ID field in the Entities list allows the user to view the Entity Details.

Entity ID - This is the Entity ID code by which the Entity is recognized within the system.

Entity Description - This is the full name of the Entity.

Limit Bank User Authorization - Allows the Bank to restrict the authorization of Bank users.

  • If “Yes”, segregation of duties exists for this entity. This means that the authorization of the Banks users are split into two types of Bank Users for the entity:

- Supervisors: Authorized to create other Bank Users as a supervisor or regular Bank user. For regular user the Supervisor must set the authorization for specific functions e.g. authorization maintenance, configuration, audit trail access, translation. The supervisor cannot carry out any of these Bank functions and define any end Users (None Bank User). 

- Bank User. Can work according to the authorization set by the supervisor. Cannot create any Bank Users, not as a supervisor nor as a regular Bank user and cannot give any regular Bank User authorization.

  • If “No”, segregation of duties does not exist for this entity.

This screen is for information purposes only and no changes can be made here.

Click  to return to the previous screen.

Add Entity

To add a new Entity, click on the  button at the bottom of the Entities list. The Entity Details form is displayed.

To complete the Entity Details form, enter the required data in the relevant fields. All the fields in this form are mandatory.

Entity ID - Enter a 3 character Entity ID by which the Entity will be recognized in the system.

Entity Name - Enter the full name of the Entity.

Copy Layout from Entity - Select the Entity from which the Layout is to be copied. The layout can be copied from any Entity that already exists in the system.

Limit Bank User Authorization - Allows the Bank to restrict the authorization of Bank users.

  • If Yes, segregation of duties exists for this entity. This means that the authorization of the Banks users are split into two types of Bank Users for the entity:

- Supervisors: Authorized to create other Bank Users as a supervisor or regular Bank user. For regular user the Supervisor must set the authorization for specific functions e.g. authorization maintenance, configuration, audit trail access, translation. The supervisor cannot carry out any of these Bank functions and define any end Users (None Bank User). 

- Bank User. Can work according to the authorization set by the supervisor. Cannot create any Bank Users, not as a supervisor nor as a regular Bank user and cannot give any regular Bank User authorization.

  • If No, segregation of duties does not exist for this entity.

 Click to save the new Entity in the System. This new Entity will automatically be assigned the number 000.

 Click to return to the Entities list without saving any changes.

Note: When a new Entity is created, the system automatically creates a General Bank User called SYSTEMXXX where XXX is the ID of the newly created Entity, using the default Bank User password. Using this ID and password, the new General Bank User can log on to the system and perform those functions for which he is authorized

Edit Entity

Click on the Edit icon to edit the Entity details. The Entity Details form is displayed.

The only field that can be edited on this form is the Entity Name. Enter a new name or change the existing name, as required.

 Click to save the changes to the Entity Name and return to the previous screen.

 Click to return to the Entities List without saving any changes.
