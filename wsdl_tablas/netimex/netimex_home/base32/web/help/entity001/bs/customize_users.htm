Customize Users

Modify User Details & Inbox Settings

Click on the Edit icon in the Users’ List Page to activate the User Modify page. This page allows the user to edit details.

User Details:

User ID - A unique description that is assigned to each user.

User Name - Full name of the User.

Default Customer - This is the customer on which the user will work on his behalf when entering the system. In order to enter any transaction on behalf of other customers, the user must return to the Choose Customer field by clicking on the Customer link on the top of each page. The Choose Customer field allows the user to switch to another customer’s transactions.

Default Language - After designating a default Language in the Languages Page of the Customize section, that language appears whenever the system is opened. If the user chooses another language as the default, the system will then appear in that language.

Supervisor - When this box is checked it means that the user has the authorization of Supervisor. A supervisor can assign authorizations to other users.

Super User - By checking this box the user may release any transaction for transmission by signing it, regardless of the signature level requirements.

Inbox Settings:

View Items - Click here to view the Inbox as top down format, item by item.

View Tables - Allows you to view the Inbox in a table.  Each Inbox item in its respective section is displayed in one line, increasing the number of items visible in the screen.

Open Incoming Correspondence – When checked this Inbox section will be open unless the Display Closed Inbox After Transaction checkbox is checked.

Open Payment/Acceptance – When checked this Inbox section will be open unless the Display Closed Inbox After Transaction checkbox is checked.

Open Signature Required – When checked this Inbox section will be open unless the Display Closed Inbox After Transaction checkbox is checked.

Open Transactions in Process – When checked this Inbox section will be open unless the Display Closed Inbox After Transaction checkbox is checked.

Display Closed Inbox After Transaction – Indicates the Inbox appearance after handling a transaction. When unchecked (default) it behaves in the original functionality and leaves the Inbox open, using the setting as defined by the other indicators. When checked, all the Inbox sections are displayed closed.

Customers Per User

Clicking on a Customer per User cell in the Users’ List Page activates the Customers per User Page. This page lists the customers for whom the user may perform transactions.

Customer - The customer name of each of the user’s customers

Signature Level - The signature level is an authorization level that indicates what transactions the customer is permitted to perform. Click on this link to see which specific transactions the user may perform for the customer

 Delete – Deletes the customer from the list of those that the user may represent.

 Edit - Allows the user to modify the signature level authorizations of the user vis a vis the customer.

    This button allows the user to add a new Customer to this user’s list.

Modify Customers Per User

Click on the Edit icon in the Customers per User page and the Modify Customers Per Users Page appears. This page allows the user to assign which level of authorization a user must have in order to perform a transaction on behalf of the customer.

 

Signature Level - This row indicates which user’s requirements for signature levels per transaction are being established.

Customer - The user may edit many customers’ transactions. This field describes for which customer these signature requirements are being set.

Inquiry Check Box - When this check box is checked, it indicates that the user is authorized to make inquiries about the customer’s transactions.

Transaction Check Box - When this check box is checked, it indicates that the user is authorized to enter transactions on behalf of the customer.

Signature Level - This field indicates what are the required signature levels for all the customer’s transactions:

  • Group A, B, C - Indicates that a User of that level authorization is required to sign the transaction.
  • Mixed Groups - Indicates that some transactions require a signature by one Group level, while other transactions require a signature by another Group level.
  • Data Entry - The user can only enter data and not change any data.
  • Not Authorized - The user cannot access this customer’s transactions.

Transaction - This column is a list of the transactions that require signatures.

Signature Level - This column is where the signature level is set to one of the above requirement levels.
