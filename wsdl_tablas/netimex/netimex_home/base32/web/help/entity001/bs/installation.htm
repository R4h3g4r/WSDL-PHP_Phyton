INSTALLATION

Installing the Web Application

1.       Start web server:

/usr/WebSphere/AppServer/bin/startServer.sh.

2.       Install NetIMEX using the browser (WebSphere tool). In the browser’s Address Bar enter the following:

URL: http://server IP: 9090/admin/
(e.g.: http://10.8.7.251:9090/admin/)

The following WebSphere Application Server screen is displayed.

3.       Enter User ID and click Submit.

4.       Select nodes, server name, and enterprise application.

5.       Click Install.

6.       Enter the relevant information and click Next. (See the following images)

Parameter: The path for the NetIMEX war file will be found on the CD supplied by Surecomp
Application Name: This will be the physical name for the application
Context Root: The URL name

Note: It is recommended to give the same name to both the Application and the Context Root.

7.      Click Next on the following Page

8.       The following page is displayed.  Click Finish.

9.    Click Finish in the previous step.

10.   From the menu on the left, select HTTP Transports.

11.   Click New.

12.       In the Port field, enter 80 and click OK.

13.       Click Exit on the menu at the top of the screen.

 

14.       On the next screen, click OK to complete the installation.

15.   Stop the Web Application Server.
        /usr/WebSphere/AppServer/bin stopServer.sh

16.    Find the HTTP Process:
        ps -ef | grepi httpd

17.    Kill the Process:
         kill –9 process id
        Example: kill -9 7324

18.   Configure the machine so that it will not automatically load the IBM http server on start up.
        Edit the web /etc/inittab file
        Find the ihshttpd string and mark the line by adding a colon at the beginning of the line.

 19.   Edit the web XML file:
        /usr/WebSphere/AppServer/installedApps/scb.ear/netimex.war/WEB-INF/ web.xml.

Note:  The section written in italics is according to the parameter specified in section 2.1 no. 6 - Application name.

Correct WORK_DIR path, if necessary.
The value of the WORK_DIR variable should the same as the Application Name parameter specified in section 2.1. no. 6 - Application name.

Example:
<param_name>WORK_DIR</param_name>
<param_value>/usr/WebSphere/AppServer/installedApps/scb.ear/netimex.war/WebServer.properties</param_value>

20.   Check the NetIMEX web properties file.
        See Appendix A NetIMEX parameters file.

21.    Start the Web Application Server.
        /usr/WebSphere/AppServer/bin/startServer.sh

Installing the Application Database Schema

1.       Login with user root.

2.       Change the directory to the NetIMEX created file system (as created in section 1.4 above).

3.       Move up to the level above the NetIMEX file system.
          cd..

4.       Untar the file from the CD-ROM.
          tar –xmvf /cdrom/netimex.arc

5.       Return to the NetIMEX file system
          cd /netimex

6.       Create the file system for the new user:
          ./crefs.bat netvan volumegr

netvan schema name
volumegr volume group

           Answer 'y' to the question.

           Note:  If you do not enter the volume group parameter, the default will be rootvg.

7.       Change the user to oracle
          su – oracle

8.       Return to netimex
          cd  /netimex

9.       Create the schema and table spaces
          ./credb.bat netvan NETVAND NETVANI

netvan Schema name
NETVAND TS for the tables (should be in uppercase)
NETVANI TS for the indexes (should be in uppercase).

 10.   Check the output log: credb.log

11.   Create the tables and primary keyes
        ./cretbl.bat netvan NETVAND NETVANI

netvan Schema name
NETVAND TS for the tables (should be in uppercase)
NETVANI TS for the indexes (should be in uppercase).

12.   Check the output log: cretbl.log

13.   Load the vanilla data from the dump file.
         ./recover.bat netvan vanilla  exp_vanilla.dmp 

netvan schema name
vanilla the vanilla  schema name (should be vanilla)
exp_vanilla.dmp dump file name  (should be exp_vanilla.dmp)

 14.   Check the output log: imp_vanilla.log

15.   Create foreign keys, indexes and triggers
        ./crefk.bat netvan 

netvan schema name

 16.   Check the output log: crefk.log

Converting the MS-SQL database to Oracle

1. Create a new Entity in NetIMEX.

2. To convert the data from MS-SQL to Oracle:

2.1  On the UNIX machine (using the UNIX emulation), from the subdirectory:

cd:\netimex run the batch file that drop triggers
./DropTriggers.bat netvan
netvan                   schema name

2.2 Check the output log: droptriggers.log

2.3 Run setup.exe from the CD subdirectory “\converting database\package”

2.4 Copy the file ms.sql from the CD subdirectory “\converting database” into the installed conversion program directory

2.5 From the installed conversion program directory, run ms2or.exe

2.6 Check the output log: ms.out

2.7 On the UNIX machine (using the UNIX emulation), run the batch file that creates triggers from the cd:\netimex subdirectory:

./cretrig.bat netvan entity
netvan              schema name
entity                entity name

3. In NetIMEX create a new language - English - in the Entity created in Step 1.

4. On the UNIX machine create a new subdirectory “\c_images\en” for the new language and copy the images files from “\c_images\bs” to the subdirectory created.
