CUSTOMIZE

The following section allows each Bank to modify this program to its own specific needs according to the categories specified in the following Customize List:

Note: The availability of options for a specific user is dependent on the user authorization level.

Preferences

NetIMEX allows the user to customize the user’s general details and configure the appearance of the Inbox. The Inbox Settings gives the user control over opening and closing the four Inbox sections by default.

By clicking on the Customize Preferences option on the Customize List, the Customize Users Form is displayed.

User Details:

User ID - A unique description that is assigned to each user.

User Name - Full name of the User.

Default Customer - This is the customer on which the user will work on his behalf when entering the system. In order to enter any transaction on behalf of other customers, the user must return to the Choose Customer field by clicking on the Customer link on the top of each page. The Choose Customer field allows the user to switch to another customer’s transactions.

Default Language - After designating a default Language in the Languages Page of the Customize section, that language appears whenever the system is opened. If the user chooses another language as the default, the system will then appear in that language.

Super User - By checking this box the user may release any transaction for transmission by signing it. The user has the authorization to make the final signature on any transaction, regardless of pending signatures usually required for the specific transaction.

Supervisor - When this box is checked it means that the user has the authorization of Supervisor. A supervisor can assign authorizations to other users.

Group Inquiries - Checking this checkbox will allow the user to run an inquiry upon all the customers that have been assigned to him. This is relevant for the following:

  • Deal Select
  • Transaction Status
  • Open Deals
  • Deal Report

Inbox Settings:

View Items - Click here to view the Inbox as top down format, item by item.

  • View Tables - Allows you to view the Inbox in a table.  Each Inbox item in its respective section is displayed in one line, increasing the number of items visible in the screen.
  • Open Incoming Correspondence – When checked this Inbox section will be open unless the Display Closed Inbox After Transaction checkbox is checked.
  • Open Payment/Acceptance – When checked this Inbox section will be open unless the Display Closed Inbox After Transaction checkbox is checked.
  • Open Signature Required – When checked this Inbox section will be open unless the Display Closed Inbox After Transaction checkbox is checked.
  • Open Transactions in Process – When checked this Inbox section will be open unless the Display Closed Inbox After Transaction checkbox is checked.
  • Display Closed Inbox After Transaction – Indicates the Inbox appearance after handling a transaction. When unchecked (default) it behaves in the original functionality and leaves the Inbox open, using the setting as defined by the other indicators. When checked, all the Inbox sections are displayed closed.

Languages

The user may choose to work in any of the languages made available by the host Bank.

 

To select the language in which NetIMEX will appear, the Bank establishes a link to the desired language when they enter the program.

When the program opens, its address is a login.asp file. To create the link to change the language in which the program appears, type ?language= plus the special 2 letter ISO language code. As long as the chosen language resides in the program database, the program will appear in that language. Other languages may be loaded through the Bank Maintenance option.

Purge

The Purge function in NetIMEX allows customers to purge old data.  Only the customer's supervisor will be allowed to use this function.  To access the Purge operation, click on the Purge option from the Customize section of the main menu.

Purge is allowed for the following objects:
  • Closed Deals
  • Rejected Deals
  • Incoming Correspondence items
  • Unsent Transactions

For each object type, you are presented with a selection criteria request.  After selecting the object type and criteria from the pull-down menu, click on the Search button.  The list of items matching the criteria will be displayed.  You can also specify to view the list within a specific date range.

Note: Items are purged physically.  You can restore the data by re-transmitting the relevant deal steps from the back office system.

The following screen illustrates the Purge function with Deals as the object type.  The Purge criteria will include Deals according to combinations of:

  • Deal Type - Identifies the type of deal.  Leave this field blank to select all the deal types.
  • Range Date - Range of expiry dates (for relevant deal types).

Select the items to purge by clicking the check box to the left of the entry (or select all the items), and request the purge function to be executed by clicking on the Purge button.

Banks

The Banks Profile page lists information pertaining to every bank that has a dealing relationship with the customer.

Note: Prior to installing NetIMEX, a batch task was initiated by IMEX which identified all the banks found in existing deals related to the customer.  These banks comprise the basic list of banks included in your installed version.  When a transaction is opened containing a new bank, the full name and address is entered. After receiving the transaction from the host, the bank ID is added automatically to the bank profile.

To view the Banks Profile window, select the Banks option from the Customize menu.

The Banks Profile window includes information on:

Bank ID - The Bank code name.

Branch - NetIMEX always displays by default the head office location of the selected Bank.

Name - The beneficiary branch name.  There may be several branches defined for a beneficiary. To define participant details for a new L/C, select a Branch code from the drop‑down list.  The full beneficiary name and address is displayed.

Street - The beneficiary branch Street name.

City - The beneficiary branch City name.

Country - The Country where the beneficiary branch is located.

Note: Bank details may only be modified by the bank’s host, although a bank may be deleted from the private bank list by highlighting a bank record and clicking the Delete icon.

Clauses

The user can create clauses or text library, which can be used to facilitate the data entry in the Text Windows on the Transaction.

The user creates the text of clauses and defines to which of the text windows and transactions each clause is relevant.

The clauses created by the user are added to those defined as standard clauses by the bank.

In the Text section of the transaction, the Open Clause Library icon  indicates that there are clauses relevant to the window. When clicking on this icon a list of defined clauses relevant for the window are displayed. After selecting the required clauses the contents of the clause is copied to the window.

Select Clause from the Customize section of the Inbox to display the Customize Clauses Search Screen.

The user has the following options:

  • Create a new clause by clicking on the Add button.
  • Search for a specific Clause by entering the Identification Code of the relevant Clause in the Clause ID field and clicking on the Search button.
  • Display a list of all existing clauses by leaving the Clause ID field empty and clicking on the Search button.

On the list of existing clauses the user can carry out the following options:

  • Edit an existing clause by clicking on the Edit icon.
  • Delete an existing clause by clicking on the Delete   icon.

How to add a clause:

1.       Click on the Add button on the Customize Clause Criteria screen to display the Clause Details Screen.

2.       Enter the Clause ID.

3.       Enter the Clause Description.

4.       Enter the Clause Text. If the text of the clause is more than 4 lines click on the Zoom Text Icon to view/edit all the data.

5.       In the Clause Availability section check the relevant checkboxes. This will indicate on which window and in which transaction the clause will be available.

6.       Click on the Save button.
