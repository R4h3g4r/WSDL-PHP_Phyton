TRANSFER LETTER OF CREDIT

There are five sections that must be completed in order to establish a new L/C.

  • General Details

  • Settlement Data

  • Participants

  • Terms

  • Text

To sign the completed deal, click the Sign icon and enter your password.

General Details

The first part of the TFR application form contains the General Details section. This section appears when you first open a new TFR.

You may also access the General Details section from anywhere in the application form. Clicking on the General Details title on any heading of the yellow Navigation Toolbar that precedes each section will bring you to this section.

To complete the General Details section, enter the relevant fields. Some of these fields are mandatory. They are indicated in Bold on the screen and by this icon  in this User Guide.

The General Details section requests the following information:

    Master L/C Number - The reference number of the original transferable Letter
of Credit.

Deal Number - The IMEX Deal number.

Transfer Available Balance - The balance of the L/C available for transfer.

    Currency - The currency of the Letter of Credit.

Note: The values in the fields above are defaulted from the IMEX Export Letter of Credit Deal.

    Transfer Amount - Enter the amount of the L/C to be transferred. This amount must be less than the available balance for transfer under the L/C (Transfer Available Balance).

   Amount Terms - Select the relevant amount terms applicable to the TFR. If vary amount/percent is selected, enter one of the following fields:

  • Varied Amount Sum

  • Varied Amount %

    Expiry Date - Enter the date on which the Transfer L/C will expire.

Place - Enter the place where the Transfer L/C will expire.

    Retain Right to Refuse AMD -  Check this checkbox to indicate that the Customer retains the right to refuse advice of amendments to the beneficiary of the TFR.

Settlement Data

The second part of the Transfer Letter of Credit application form contains the Settlement Data section. Scroll down from the General Details to access this section.

You may also access the Settlement Data section from anywhere in the application form. Clicking the Settlement Data title on any heading of the yellow Navigation Toolbar that precedes each section will bring you to this section.

The second part of the Import L/C application form contains the Settlement Data information of theTFR L/C and the following information is required:

Days for Presentation - Specifies the number of days following the date of the transportation documents that the documents can be presented for payment.

    Available With - Specifies which bank is authorized to negotiate the documents on behalf of the beneficiary. 

   Settlement By - The method of settlement is defaulted according to the orginal L/C and this field cannot be changed.

The following shipment information is may be entered:

Earliest - The earliest permitted date for shipping

Latest - The latest permitted date for shipping

Participants

The third part of the TFR application form contains the Deal Participants section. Scroll down from the Settlement Data section to access this section.

You may also access the Deal Participants section from anywhere in the application form. Clicking on the Participants title on any heading of the yellow Navigation Toolbar that precedes each section will bring you to this section.

The Participants form enables you to define the Beneficiary and Advising Bank.

Note: The initial NetIMEX installation includes a list of all the Parties and Banks which were included in previous transactions with the Bank.

The following fields appear in the Participants section:

Beneficiary and Advising Bank Details - Specifies the Beneficiary and Advising Bank name and location details relevant to the TFR. Enter the Beneficiary or Advising Bank details in the respective blocks. The user may select a Beneficiary or Advising Bank by entering a code number or by filling in the name and location details. If the code is entered in the database, the name and location details appear automatically when the code is keyed in.  If the required Beneficiary or Advising Bank code does not exist in the database, leave the code field blank and type the full name and address of the Beneficiary or Advising Bank in the fields provided.

Search by Code - The database codes that identify the Beneficiary or Advising Bank. The user can identify a specific Party or Bank by entering any number of characters in the Search by Code field and then moving to another field e.g. Enter B and the details of the first Party/Bank that starts with B will be displayed.

Search by Name - The database names of the Beneficiary or Advising Bank.

Name - The beneficiary or Advising Bank branch name.  There may be several branches defined for a beneficiary. To define participant details for a new L/C, enter the full Beneficiary or Advising Bank name.

Street - The Street name where the Beneficiary or Advising Bank is located.

City - The City where the Beneficiary or Advising Bank is located.

Country - The Country where the Beneficiary or Advising Bank is located.

Default Bank: When a specific Beneficiary is entered in the Participants section, the Advising Bank of the last deal using that Beneficiary together with the currency of that last deal will automatically appear in the Advising Bank fields as the Default value.

Note: The Beneficiary details are always mandatory.

Additional Parties

The number of Additional Parties sections may differ depending on configuration. Each section enables the user to enter information about an additional Party associated with the transaction.

To define a Party:

1.       Select the Party Type.

2.       The system will display the relevant Codes for the select Party Type

 The following fields appear in each Additional Party section:

Additional Party Details - Enter the Party details in the respective blocks. The user may select a Party by entering a code number or by filling in the name and location details. If the code is entered in the database, the name and location details appear automatically when the code is keyed in.  If the required Party code does not exist in the database, leave the code field blank and type the full name and address of the Party in the fields provided.          

Search by Code - The database codes that identify the Party. The user can identify a specific Party by entering any number of characters in the Search by Code field and then moving to another field e.g. Enter B and the details of the first Party that starts with B will be displayed.

Search by Name - The database names of the Party.

Name - The Party branch name.  There may be several branches defined for a beneficiary. To define participant details for a new transaction, enter the full Party name.

Street - The Street name where the Party is located.

City - The City where the Party is located.

Country - The Country where the Party is located.

Terms

The fourth part of the TFR application form contains the Terms section. Scroll down from the Participants section to access this section.

You may also access the Terms section from anywhere in the application form. Clicking on the Terms title on any heading of the yellow Navigation Toolbar that precedes each section will bring you to this section.

Select the appropriate response for each term applicable to the Transfer L/C:

  • Confirmation

  • Partial Shipment

Texts

The Text section, which is the last part of the Transfer Letter of Credit application form, allows the user to enter additional customized information and requirements to the TFR. Scroll down from the Terms section to access these fields.

You may also access the Texts section from anywhere in the application form. Clicking on the Text title on any heading of the yellow Navigation Toolbar that precedes each section will bring you to this section.

The Text section enables you to:

  • Enter free text to specific categories

  • View and specify account details

Additional information can be entered for the following categories and may include other custom-defined categories:

Comments - Used to enter additional comments.

Customer Comments - Used to enter additional customer instructions.

Zoom Text Icon  - Click on this icon to enlarge the text box area. You may edit or enter new data in the enlarged text box.

Open Clause Library Icon  - When clicking on this icon a list of defined clauses relevant for the window are displayed. After selecting the required clauses the contents of the clause is copied to the window. 

Clause Library
In order to display the clauses for a specific text window click on the Open Clause Library Icon next to the relevant window.

On the Clause Library window select the clause/s that should be entered into the text by checking the relevant checkbox and clicking on the Select button. In order to include all the Clauses listed in the Clause Library check the checkbox at the top of the column and click on the Select button.

When returning to the Text Screen the user can view the detailed description of the Clause which will be included in the Text. In order to remove a clause from the Text the user must delete the text of the clause from the relevant window.

Defined Clause Descriptions
An additional option exists to display clauses as defined in the clauses form, which the user can select by selecting the desired checkboxes. This replaces the use of the Clause Library and free text windows.

There are two options for this function, which is established during bank setup:

  • The user can select more than one clause from one text window.

  • The user can select only one clause from one text window.

In a text window which is marked as mandatory, a validity check is performed to ensure that at least one clause is selected.

Account Details

The Customer Account Instructions Window enables you to instruct the bank how to make payment for various TFR items such as the documents, bank commissions and charges. 

Up to three Bank Charges can be defined. If data exists in the Payment Item field a corresponding Account number must be entered. If there is a blank Payment Item field, when relevant, any additional Payment Item and corresponding Account number may be selected.

To specify the customer’s account details, select the appropriate field on the Account Instructions Form. The Bank Charges describes the payment item to be paid, for example, Principal Amount, Charges. The Account Number selected from the drop-down list is from the Customer Profile. Use the Additional Instructions window to add additional payment instructions.
