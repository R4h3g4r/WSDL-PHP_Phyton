ENTITIES

Introduction

The NetIMEX Multi-Entity feature is designed to allow a Bank using NetIMEX to host the electronic Trade Finance operation of several Entities in one installation of the system, rather than having several different installations - one for each entity.

The appearance for each Entity created is copied from an existing Entity.  The User creating the entities can select the Entity from which to copy the appearance of the screens in the new Entity. All the necessary tables, images, help files and other information required by the system is automatically copied to the new Entity on its creation.

NetIMEX is supplied with one General Bank User defined. This user can perform only two actions in the system:

  • Maintain General Bank Users. These users will be able to perform these same two functions only.
  • Create new Entities.

A customer User will automatically be logged in to the Entity to which he belongs, and will be able to view and operate only those Customers belonging to his group, according to the authorization given to him. The regular Bank user has the option to edit, create and customize certain features for his Entity. For further details please refer to the section on Bank User Work Options with Entities.

Customizing Entities

The first screen to which the General Bank User is routed after login is the Customize Entity screen, where a list of all Entities is displayed and Entities can be created, edited or deleted.

Entity ID - This is the Entity ID code by which the Entity is recognized within the system.

Entity Description - This is the full name of the Entity.

Limit Bank User Authorization - Allows the Bank to restrict the authorization of Bank users.

  • If Yes, segregation of duties exists for this entity. This means that the authorization of the Banks users are split into two types of Bank Users for the entity:

- Supervisors: Authorized to create other Bank Users as a supervisor or regular Bank user. For regular user the Supervisor must set the authorization for specific functions e.g. authorization maintenance, configuration, audit trail access, translation. The supervisor cannot carry out any of these Bank functions and define any end Users (None Bank User). 

- Bank User. Can work according to the authorization set by the supervisor. Cannot create any Bank Users, not as a supervisor nor as a regular Bank user and cannot give any regular Bank User authorization.

  • If No, segregation of duties does not exist for this entity.

Delete  - Clicking on this icon will erase the Entity from the system.

Note: An Entity can only be deleted from the system if there are no regular customers for tlis Entity defined in the system.

Edit - This function allows the user to make changes to the Entity Description.

 Clicking on this button allows the user to create a new Entity,

 Returns the user to the previous screen without making any changes.

General Bank Users List - Click on this link to display the list of all General Bank Users in the system.
