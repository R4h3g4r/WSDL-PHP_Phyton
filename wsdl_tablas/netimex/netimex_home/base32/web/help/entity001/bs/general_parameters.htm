General Parameters

To update the NetIMEX general parameters, select General Parameters from the Banks Maintenance option, in the Customize section of the main menu.

Working Under a Frame

NetIMEX automatically detects if the application is operating within a frame. If working under the Bank’s frame the Bank has the following options:

  • Header: Check to include the NetIMEX Header GIFs in the NetIMEX pages.
  • Header Links: Check to include the Links GIF (bottom GIF) of the header in the NetIMEX pages.
  • Home Button: Check to activate the Home Link in the NetIMEX header.
  • Disable User Links: Check to deactivate the User Link in the NetIMEX header

Logoff Option

The General Parameters form defines whether the logoff option is used or not. Check the Logoff Button checkbox in order to use the logoff option.

An option in the NetIMEX footer allows you to explicitly request logging off from the session. The logoff option can be found on the bottom of any screen enabling you to exit at any time. Any attempt to access a NetIMEX page after logging off will route you back to the login page. 

Changing Bank Links

The NetIMEX footer contains links that may be customized by the Bank within the General Parameters Form.

The links are as follows:

Home Page Name Defines which IMEX page will be the NetIMEX homepage.  The user can bypass the homepage and go directly to the page of your choice (i.e. Inbox) by typing the name in the "Home Page Name" parameter.
What's New Specifies the page (form) to which NetIMEX will link when the What's New option is selected.
Feedback Specifies the page (form) to which NetIMEX will link when the Feedback option is selected.
E-Mail Specifies the page (form) to which NetIMEX will link when the E-Mail option is selected.
Company Name Allows the user to specify a company name.

Transactions Per Page

The number of transaction per page can be determined (it can be less than one).

The pages that need this parameter are: Amend Letter of Credit, Amend Guarantee, Deal Message List, Transaction Status, Clause List, Deal Select and Copy From List.

Generate Deal Number

The user can determine when to generate the Deal Number in one of the following ways:

  • Not At All
  • After Last Signature
  • After Any Signature
  • In Any Case

Allow Collection Letter Printing

The Collection Letter Printing feature allows the Bank to configure when the user can generate the Collection Letter for direct Export Collections.

One of the following options must be selected to indicate when or if the Collection Letter will be generated:

  • Not At All: Will not be generated
  • After Last Signature: When all signatures that are required have been entered
  • After Any Signature: At least one signature is required

Allow Cover Letter Printing

The Cover Letter Printing feature allows the Bank to configure when the user can generate the Cover Letter for regular Export Collections. The Cover Letter may be generated, printed and attached to the set of documents sent to the Bank.

One of the following options must be selected to indicate when or if the Cover Letter will be generated:

  • Not At All - Cannot print
  • After Sign - Can only print after all the required signatures have been entered
  • After Save - Can print after save even prior to final signature.

Allow Printing Items

This feature allow the Bank to specify when or if the user can print specifically tailored applications form for Letter of Credit, Amendment Letter of Credit and/or Remittance as follows:

Not At All - Cannot print

After Sign - Can only print after all the required signatures have been entered

After Save - Can print after save even prior to final signature.

Copy From

Selection of the relevant radio button on the General Parameters screen will determine what is displayed in the Copy from results:

  • Customer side
  • Bank side
  • Both sides

Generate Reference Letter

When the Generate Reference Number checkbox is checked, if the user indicates that the Reference Number is not required, the system will automatically generate the Deal Identification Number.

Disable Bulk Signatures

The user is allowed to sign a group of transactions within the specific authorization levels. If this checkbox is checked the Bulk Signatures function will be disabled and the user will not be allowed to sign a group of transactions.

Disable Bulk Signing of Transactions in Process

The user is allowed to sign a group of transactions in process within the specific authorization levels. If this checkbox is checked this function will be disabled and the user will not be allowed to sign a group of transactions in process.

Disable Bulk Payment / Acceptance

The user is allowed to process a group of payment and acceptance steps within the specific authorization levels. If this checkbox is checked this function will be disabled and the user will not be allowed to process bulk payment and acceptances.

PO Integration Enable

Check the checkbox in order to indicate that NetIMEX is connected to an external portal to receive Purchase Orders.

Open Help Page In New Window

Check if NetIMEX help (after clicking on the HELP button) should be displayed on a new browser window.

Open Incoming Correspondence in New Window

Check to display Incoming Correspondence in a separate browser window.

Release Period For Locking Transaction

Enables a user to lock a transaction for a defined period of time, between 0 - 9999 minutes.
The default value of the release period is ‘0’.
If the release period is ‘0’ only the user that locked the transaction can open it.
If the release period is, for example, ‘10’, the user has placed a lock on the transaction for 10 minutes, and other users cannot edit the transaction for that period of time. After 10 minutes, the transaction is automatically unlocked, enabling all users to edit the transaction.

Restrict Supervisor Authorization

Check the checkbox in order to allow the user to restrict the authorization of customer supervisors as follows:

Supervisor cannot process transactions. Authorization of supervisors for transaction is always “Not Authorized”.

Supervisors cannot sign transactions; therefore supervisors cannot be given an attribute of super user.

Import New Remittance

When checked indicates that Remittance will be displayed on the Menu.

Regional Settings

Allows the Bank to define specific regional settings for each entity as follows:

  • Decimal Symbol
  • Grouping Symbol
  • Date Delimiter
  • Date Format

Number of Digits After Decimal Point

Allows the Bank to define in the Inquiries how many digits to display after the decimal point in the Amounts. It can be set in either of the following ways:

  • According to Currency
  • Two Digits
