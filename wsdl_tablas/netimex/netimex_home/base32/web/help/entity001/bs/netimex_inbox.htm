NETIMEX INBOX

Overview

The Inbox is divided into two main areas.

Situated on the left side of the page, the user will find the Main Menu. From this menu, the user can perform all the main functions of the system. For example: Open New Deals, Go to Inquiry, Customize the System, Build New Templates.

The right side of the Inbox Page is divided into four sections:

  • Incoming Correspondence

  • Payment/Acceptance

  • Signature Required

  • Transactions in Process

The user has control over opening and closing the 4 Inbox sections. By clicking on the picture on the left hand side of the section heading it will open or close the section depending on the status. When the section is open the picture will be clear and when it is closed it is dimmed.

Note that each section is handled separately. After clicking on a specific section heading the Inbox will be re-displayed with this section’s new settings.

After saving or signing a transaction the user is returned to the appropriate Inbox (Import or Export). This causes the Inbox items to be retrieved from the NetIMEX database and re-displayed. When returning to a closed Inbox after transaction processing, the Inbox is displayed according to the section settings but no items are displayed even for open sections. In this case the user will not be able to open or close the sections. Only after re-entering the Inbox via the Inbox, Import or Export menu options the Inbox sections can be opened or closed.

Configuring Inbox Settings

The user can define which of the sections should be opened when entering the Inbox by using the Inbox Settings. These settings can be defaulted by selecting the Preferences option in the Customize section on the main menu.

When a section heading is checked it indicates that the relevant section will be opened by default.

The Displayed Closed Inbox After Transaction governs the Inbox after handling a transaction. When unchecked it behaves in the original functionality and leaves the Inbox open, using the setting as defined by the other four indicators. When checked, all Inbox sections are displayed closed.

Note: This page is also displayed during configuration by the Supervisor when clicking on the User update link from the Users menu option. In this way when the Supervisor configures the user profiles, the Inbox settings can be set according to the initial bank preferences.

Incoming Correspondence

In this section the user can find all the documents that arrived from the bank.

Next to each document there are two icons.  These icons allow the user to print or remove the document from the arrival list. (This action normally takes place once the user is satisfied that the documents are no longer required to be held on the arrival list.)

Bulk Operations

The user can perform print and delete operations on a number of selected Incoming Correspondence items without the need to request the operation one item after another (performing the operation in Bulk).  By clicking on the Incoming Correspondence section header (the yellow banner), the Bulk Operations page is displayed.  The user is presented with a selection criteria request.  After selecting the criteria, click the Search button.  The list of Incoming Correspondence items matching the criteria will be displayed.  Select the items to work on by clicking the check box to the left of the entry (or select all the items), then select the required operation (print or delete).  The bulk operation will be performed once the required function has been executed.

You can select items by Transaction Name, Deal Type, Deal No., and within a specific date range.

Payment / Acceptance

In this section the user may view all the information relating to the payments and acceptances requiring either authorization or rejection.

Next to each Payment, the user will find three icons that are used for Establishing Payment Transaction, Rejecting Payment Transaction and Printing Payment Transaction instructions.

Signature Required

This section lists all the transactions that require a release signature prior to their transmission back to the Bank.

If the user is not authorized to sign transactions or if the user is not defined as a supervisor, not all the icons will be displayed.

The following icons may be displayed next to each transaction:

  • View Signatures: allows the user to see who has already signed the transaction and who has yet to sign.

  • Sign: (if active) allows the user to sign the transaction.  Prior to actually signing the user is asked for a security password.

  • Reject: allows the user to reject the transaction.  If the user does reject the transaction, all previous signatures for this transaction are deleted and the transaction reverts to “Transactions in Process” status.

  • Edit: Allows the user to edit the transaction.  If a transaction is being edited for a second time, the user will receive a warning that all previous signatures are about to be deleted.  Should this happen, the transaction will revert to “Transactions in Process” unless the user re-signs the edited transaction once again.

  • Print: Allows the user to print the current form.

Bulk Signature

The user can sign a group of transactions (within the specific authorization levels) without the need to sign each item individually.

How to sign a group of signatures:

  1. Click on the Signatures Required Section Header; the Bulk Signature Criteria Screen is displayed.

  1. On the Bulk Signature Criteria screen the user selects the relevant group of transactions waiting for signature and clicks Search. If the user does not enter a value into any of the fields then, after clicking Search, a list of all the transactions waiting for signature is displayed.

  1. The Sum Total is a total of all the checked transactions on the list with the same currency. If more than one currency is checked no Sum Total is displayed.

  2. In order to check all the transactions check the checkbox at the top of the column.

  3. Click on the Sign button; a confirmation screen is displayed giving a summary of the transaction details.

  1. Enter your password and click on the Sign button.

  2. The signature process has been performed.

Transactions in Progress

The Transaction in Progress Form, allows the user to view all incomplete transactions that are open and require either a signature or the entry of additional data. 

The Inbox page, which is easy to navigate, allows the user to easily “jump” from one page to another using the navigating toolbars on the top of the page and above each section of the page.

Entering Payment Instructions for Letter of Credit Deals

The payment instructions for L/C give the bank specific instructions on how to debit accounts when the payment method is Sight.   This is relevant in cases where the bank requires customer approval for a payment for documents negotiated under an L/C for example, discrepant documents.

Select a deal from the Payment / Acceptance section of the Inbox.

To specify payment instructions, select a deal and click the Open icon.

Note: The Drawing number for a deal record indicates the payment number.

Defining Payment Instructions

The Payment Instructions Window displays payment details for the selected L/C.  Since the L/C has been issued, the reference number, deal number and payment details are protected.  The Payment Instructions Window enables you to:

  • Provide payment instructions including the modification of account details.

  • Make a loan request.

  • Reject the documents, stating reasons.

  • Include additional payment instructions.

Defining Account Details

The Customer Account Instructions Window enables you to instruct the bank how to make payment for various L/C items, such as the documents, bank commissions and charges.

Up to three Bank Charges can be defined. If data exists in the Payment Item field a corresponding Account number must be entered. If there is a blank Payment Item field, when relevant, any additional Payment Item and corresponding Account number may be selected.

To specify the customer’s account details, select the appropriate field on the Account Instructions Form. The Bank Charges describes the payment item to be paid, for example, Principal Amount, Charges. The Account Number selected from the drop-down list is from the Customer Profile. Use the Additional Instructions window to add additional payment instructions.

Defining Additional Information

This section includes additional details of the transaction. If a field header is bold it indicates that the field is mandatory and the user must enter the relevant text in the corresponding data entry field. If the field header is not bold it is not mandatory and the user does not have to enter data.

L/C Loan Request

To request the bank to finance payment of a deal rather than paying for it outright, you can apply to the bank for a loan.

The following list describes the fields in the Loan Request Form:

Amount - Specifies the loan amount.

Ccy - Specifies the loan currency.

Loan Type - Specifies the type of loan being requested.

Value Date - Specifies the date the loan becomes effective.

Period - The length of the loan, in days.

Maturity - Specifies the repayment date of the loan if the period is not defined.

Making a Loan Request

To make one or several loan requests:

1.       Enter loan request amount.

2.       Enter the currency code for the loan.

3.       Select the loan type from the drop-down menu.

4.       Enter the value date.

5.       Either enter the loan period (in days), or the date of maturity.

6.       Use the free format text window to add additional loan request details.

Reject Payment Details

In case of conforming documents, the record does not appear in any of the worklist windows since the issuance application infers an undertaking to pay against conforming documents.  In case of discrepant documents, there is an option to reject the documents, unless a shipping guarantee has already been issued.

To reject the documents, click the Reject option.  Use the free format text window to explain the reason for rejecting the documents.

Free Format Window

The Free Format Text window allows for the inclusion of additional details related to the transaction.  For example, should the customer reject the payment, the text window is used to explain why.

Entering Acceptance Advice for Letters of Credit Deals

In the case that the L/C was issued on a Time basis, and the bank requires your approval for the documents (for example, the existence of discrepancies), the record appears in the Payment / Acceptance section of the Inbox.  The acceptance instruction enables you to approve the payment, providing debit instructions, or reject the documents, providing reasons.

To specify acceptance advice details, select a deal and click the Open icon.

Defining Acceptance Advice

The Acceptance Advice Window displays payment details for the selected L/C. Since the L/C has been issued, the reference number, deal number and acceptance details are protected. The Acceptance Advice Window enables you to:

  • Provide payment instructions including the modification of account details.

  • Make a loan request.

  • Reject the acceptance advice, providing reasons.

  • Include additional advice instructions.

Note: Since this is a Time L/C, any instruction provided at this stage is referred to the payment, which is made at the maturity date.

Defining Account Details

To specify account details, select the appropriate field and select the information from the drop-down list.

L/C Loan Request

To request the bank to finance payment of a deal at maturity rather than paying for it outright, you can apply to the bank for a loan.

Entering Payment Instructions for Collection Deals

When the bank receives documents for Collection on Sight basis or Time (and the draft has been signed), the deal appears in the Payment / Acceptance section in the inbox.  This enables you to authorize the bank, via NetIMEX to make payment for the shipping or financial documents and instruct the bank as to the method of payment.  Authorization can also be made for Clean Collections.  Clean collections means that only financial documents (drafts, bills of exchange, etc.) are included in the transaction.

Note: For documents on Time basis, NetIMEX is not used to convey instructions since you are required to sign and return the physical draft.  Time means that the value of the collection deal is payable on a specified date (Due Date), or after a period of time (Tenor Days) and some condition has been fulfilled (Draft After).

Collection deals are received from the bank’s host requiring payment instructions. Payment instructions include account details, loan requests and additional payment instructions, with the ability to modify the payment amount and payment date.

Select the deal from the Payment / Acceptance window in the Inbox section.

Defining Payment Instructions for Collection Deals

The Payment Instructions Window displays details for the selected Collection.  The Payment Instructions Window enables you to:

  • Define account details.

  • Make a loan request.

  • Reject the documents.

  • Include additional Collection instructions.

The following list describes the fields in the Payment Instructions Window that are open and can be modified:

  • Amount - Enables you to modify the collection amount (e.g. to effect partial payment).

  • Payment Date - The date the collection is payable.

  • Final Payment - Enable the check bullet to indicate that this is the final payment.

Defining Account Details for Collection

The Customer Account Instructions Window enables you to instruct the bank how to make payment for various Collection items such as the documents, bank commissions and charges. 

Up to three Bank Charges can be defined. If data exists in the Payment Item field a corresponding Account number must be entered. If there is a blank Payment Item field, when relevant, any additional Payment Item and corresponding Account number may be selected.

To specify the customer’s account details, select the appropriate field on the Account Instructions Form. The Bank Charges describes the payment item to be paid, for example, Principal Amount, Charges. The Account Number selected from the drop-down list is from the Customer Profile. Use the Additional Instructions window to add additional payment instructions.

Rejecting a Collection

To reject the documents, click the Reject option.  The Payment and Loan Details Windows become protected.  Use the free format text window to explain the reason for rejecting the documents.

Free Format Window

To view the documents received from the bank, which are related to the payment request, click the Incoming Documents icon from the toolbar.  To view a document, select a document record.  The text is displayed below the document list.

The Free Format Text window allows for the inclusion of additional details related to the transaction. For example, should the customer reject the payment, the text window is used to explain why.
