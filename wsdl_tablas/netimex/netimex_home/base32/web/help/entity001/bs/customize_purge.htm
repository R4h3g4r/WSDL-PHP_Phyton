Purge

The Purge function in NetIMEX allows customers to purge old data.  Only the customer's supervisor will be allowed to use this function.  To access the Purge operation, click on the Purge option from the Customize section of the main menu.

Purge is allowed for the following objects:
  • Closed Deals
  • Rejected Deals
  • Incoming Correspondence items
  • Unsent Transactions

For each object type, you are presented with a selection criteria request.  After selecting the object type and criteria from the pull-down menu, click on the Search button.  The list of items matching the criteria will be displayed.  You can also specify to view the list within a specific date range.

Note: Items are purged physically.  You can restore the data by re-transmitting the relevant deal steps from the back office system.

Purge - Deals

The following screen illustrates the Purge function with Deals as the object type.  The Purge criteria will include Deals according to combinations of:

  • Deal Type - Identifies the type of deal.  Leave this field blank to select all the deal types.
  • Range Date - Range of expiry dates (for relevant deal types).

Select the items to purge by clicking the check box to the left of the entry (or select all the items), and request the purge function to be executed by clicking on the Purge button.

Check the checkbox at the top of the first column to select all the entries in the list. 

AZ or ZA allows you to sort the data in an ascending or descending order.

Deal No. - Identifies the Deal number.

Reference No - A unique deal identification number.

ISS/EFF Date - The issuance or effective date of the deal.

Mat/Exp Date - Specifies the maturity or expiry date of the deal.

Counter Party - Identifies the counterparty.

 

Purges the selected entries in the list.

 

Returns to the previous screen without making any changes.

Purge - Incoming Correspondence

The following screen illustrates the Purge function with Incoming Correspondence as the object type. The Purge criteria will include Incoming Correspondence according to combinations of:

  • Text Type - Identifies the text type.  Leave this field blank to select all the text types.
  • Range dates - Range of transmission dates.

Transaction Name - Name of the transaction.

Deal No. - Identifies the Deal number.

Reference No - A unique deal identification number.

Drw. No. - Identifies the drawing number.  The Drawing Number indicates how many actions have been performed on the deal application process.

Text Type - Identifies the selection criteria.

Currency/Amount - Specifies currency / amount of the deal, if any.

Date - Specifies the correspondence date.

 

Purges the selected entries in the list.

 

Returns to the previous screen without making any changes.
