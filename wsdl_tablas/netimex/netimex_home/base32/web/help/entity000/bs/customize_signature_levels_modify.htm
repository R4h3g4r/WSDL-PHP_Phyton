To Modify Signature Levels

The supervisor may modify signature requirements if there are no outstanding transactions of that variety for that customer. Fill in the fields to add new signature requirements. This page is identical to the Modify Signature Requirement Page.

Transaction Name - These are the open types of transactions in the system. Choose the transaction type to whose requirements you wish to add.

Up to Amount - This is the limit authorized for the signature requirement. Set a limit up to which these signature requirements will be valid.

Group A / Group B / Group C  - These groups represent different authorization levels. This list states how many signatures must be obtained from each group at specific deal amounts in order to authorize the transaction.

    This button enters the data to the database. After this the entry will appear in the Transactions in Process section of the In-Box.

    This button undoes the previous step.

Signature Level Setting Level and Validation

When signing transactions for signature levels, the user should consider the defined local currency, relative to the system local currency and transaction currency.

Notes:
1. When changing the customer local currency, the system will not recalculate the signature level matrix.
2. The system is unable to change the customer local currency, if there are transactions waiting for a customer’s signature.
