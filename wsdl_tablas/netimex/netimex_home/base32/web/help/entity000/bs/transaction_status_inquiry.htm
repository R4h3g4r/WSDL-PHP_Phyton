Transaction Status Inquiry

The first Inquiry option is the Transaction Status Inquiry. This option enables the user to view a list of customer initiated transactions  The Transaction tracks:

  • Viewing details pertaining to the signatures required for the current transaction.
  • Viewing a list of the signatures that have signed a transaction.
  • Tracking changes in the transactions.

To view the Transaction Status Request Form, select the Transaction Status option from the Inquiry menu or enter details in the Transaction Status request form that appears when you open the Inquiry section.

Status inquiries are requested according to the following parameters:

  • Transaction: This section of the Inquiry describes factors relating to the transaction itself.
  • Status: Since each change affects a transaction’s status, this report lists when the transaction changed to the current status.

The transaction section requires that two fields are entered:

Status – This field limits the search to the current transaction stage. Thus, transactions may be identified by what actions were performed on them or in light of what stage they were assigned, for example: In process, Released, Signature Required etc.

Name – This field identifies the transaction deal type and step.

The Status section requires that a further two fields are entered:

Date from/to - The date parameter restricts the search to a range of when the transaction changed to the current status.

All Customers - Check this box if the request is for a group of Customers or only for the Customer the user is currently working with.

Note: Availability of this field is dependent on the user having permission to activate inquiries at the group level.

Save Criteria - Check this box for NetIMEX to save the inquiry criteria as the default display in the user profile. All subsequent requests for the inquiry will use the saved criteria as the default.

If the user wants to remove a saved criteria status for the next default Transaction Status, thereby creating an empty criterion status, the following step needs to be taken:

  • All fields need to be blank, except the Save Criteria field, which need to be checked, and then click Search.
A user is allowed to use one criterion per inquiry.

 

Click on this button to begin the search. A list of transactions that conform to the retrieval criteria appear in the Inquiry Results Table as shown in the next screen.
When the user clicks on this button the list of inquiry results are exported to Excel, which will automatically be invoked from the user’s browser. The user can then manipulate the downloaded data, including saving it.

Inquiry Transaction Status Results Table

The top of the report lists the search criteria that defined the report parameters. The bottom of the page shows the search results.

 

 

After receiving the inquiry results, the user may sort them by any of the Inquiry Results table column options.

To sort the Inquiry Results Table:

  • Click on the   button for ascending sort order or
  • Click on the   button for descending sort order
  • Then select any column with a blue heading.

To view a transaction:

  • Double click on a transaction record in the Transaction Name column in the Transaction Status Search Results Table.

To view signatures:

  • Click on the signatures icon   to view the Signature Details table and the list of those who have signed the transaction.

 

The following information is provided in the Signature Requirements Table:

Signature List: The Signature List provides details about the signatures entered in the Deal Request form. The Signature List provides the following information:

  • User Name – Indicates who signed the form.
  • Level – Describes the authorization level of the user who signed the form.
  • Date/Time – Tells when the signature was entered.

Required Signature: These fields provide information about the signature requirements for this form. The Required Signature fields provide the following information:

  • Level – This indicates what level officer must sign the form
  • Number – This indicates the number of Users from that level whose signature is required on the form.

To print the Collection Letter:

  • The print icon   will appear on the Direct Collection transaction when all criteria enabling the printing of the collection have been met. Click on this icon to print the Collection Letter.
