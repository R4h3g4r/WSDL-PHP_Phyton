User Authorization / User Disable Option

This page indicates whether or not the user is a supervisor and provided with the level of authorization that it entails.  Click on the User-id link or the pencil icon in the User area on the User and Customers Details Form  to enter this screen.  NetIMEX will not allow a user that signed transactions to be deleted so his details will remain in the system for the purpose of auditing the signatures, therefore it is also possible to disable a user.  Check the Disable box on the user profile screen to flag the user as disabled.  As a result, disabled users will not appear in the Supervisor's user maintenance function.

Note: Disabling a user in NetIMEX should be performed in parallel to revoking the user's access rights in the bank's security system.

User Details With Segregation of Duties

Click on the "Authorization" link to enter this screen. When Segregation of Duties exists for this entity the authorization of the Bank users are split into two types of Bank Users for the entity:

1.      Supervisor Bank User

A supervisor has the following authorizations:

  • Authorized to create, delete or edit any Supervisor Bank Users or Regular Bank Users. For Regular Bank Users the Supervisor must set the authorization for specific functions e.g. configuration, audit trail access, translation.
  • Can change a Regular Bank User to become a supervisor Bank User by setting the supervisor checkbox. The original Bank User permissions will be deleted.
  • Can only perform specific functions, i.e., all items under Inquiry, Customize Preferences, Languages, Users, Customers, Bank Maintenance – Group only. When a Supervisor Bank User logs in to NetIMEX only these functions will be available.
  • Does not have permission for transactions therefore the signature level for the Bank is “Not Authorized”.
  • The supervisor cannot carry out any of these Bank functions and define any Customer Users.

2.      Regular Bank User

A regular Bank User can work according to the authorization set by the supervisor. Cannot create any Bank Users, not as a supervisor nor as a regular Bank user and cannot give any regular Bank User authorization.

User Details – Defines the User ID and Name

Supervisor – When checked indicates that the user is a supervisor. Even if the permissions are changed to allow they will saved as not allowed.

Disable – Check the Disable box to flag the user as disabled. As a result, disabled users will not appear in the Supervisor's user maintenance function. Disabling a user in NetIMEX should be performed in parallel to revoking the user's access rights in the bank's security system.

Permissions – The following permissions can be given to the Customer Service or Bank User Functions:

  • Allow: Allow to access and update
  • View: View only, allowed to access but not update
  • Not Allow: Not authorized, no access to the function

Only a Regular Bank User can be given permission for these functionalities:

Change Customer – In Edit/Create/Delete Mode the Regular Bank User can do the following for the bank customer user: Transaction inquiry, View signature level, Create/Update templates, View banks, Handle parties, Clauses and Purge. In View Mode the Regular Bank User can do the following for the bank customer user: View Banks, Parties, Clauses, Purge, Transaction inquiry, signature level, templates (cannot create/edit/update templates, will not see any transactions from the menu and in “Customization Templates” the edit and delete icons will be disabled).

User Management – (Group, delete add update view, Group by customer, Customer per group, Delete/Update/View Customer and User). Only in view mode the user will see all the relevant pages. In Edit/Create/Delete Mode Bank users that do not have the permission will not be able to see the group in the bank maintenance page.

Translations – (New Language, Delete Language, Tables Translation, Titles Translation). In view mode it is only relevant for table translation and title translation. In Edit/Create/Delete Mode Bank users that do not have the permission will not be able to see the languages related in the bank maintenance page.

Configuration – (General Parameters, Deal Number for Department, Help Page Configuration, Signature level, Clauses, Customize template). In Edit/Create/Delete Mode Bank users that do not have the permission will not be able to see the configuration pages in the bank maintenance page

Audit Trail – In Edit/Create/Delete Mode Bank users that do not have the permission will not be able to see the audit trail page in the bank maintenance page.

PO Integration – In Edit/Create/Delete Mode Bank users that do not have the permission will not be able to see the 'PO Integration page in the bank maintenance page.
