Bank Maintenance

Bank Maintenance is a function that is reserved for the Bank only to use in adapting and updating the functions to its specific requirements. Clicking on each topic link accesses each function.

The following areas can be edited in this section:

Group – Different Customers can be related by an umbrella organization known as a group. This section allows the user to view or edit the lists of customers and their super organizations.  

Group by Customer ID – This search field provides an alternative way of finding out to which group a customer belongs.

New Language – This function allows the Bank to define a new language in the system.

Delete Language – This function is used to delete a language in the system.

Tables Translation – The tables in the system originated in IMEX. This function translates IMEX tables to compatible formats in NetIMEX form.

Titles Translation – This function allows the Bank to edit any of the headings of program.

General Parameters – This function allows the Bank to customize different parameters that appear on each page.

Deal Number for Department – This function configures the deal number generation.

Help Page Configuration – This function specifies a Help page or Help application for each NetIMEX page different than the Help page delivered with NetIMEX making it unnecessary to override the supplied Help pages with the Bank’s Help pages.

Audit Trail – This function allows the Bank to configure activity logging and generate an inquiry regarding the activity in NetIMEX.

PO Integration – This function allows NetIMEX to perform operations related to acceptance of Purchase Orders from an external portal.

Set Up Dynamic Fields – This function allows the Bank to configure which of the Dynamic Fields for each transaction will appear as mandatory fields in NetIMEX.

Group

A Group consists of a super organization that may contain several subdivisions that we define as Customers. Users belong to Groups and are authorized to enter transactions at the Bank on behalf of specific Customers.

Click on the Group link on the Bank Maintenance Option Page to display the list of Groups and their customers. In this section the user can define and edit the groups, and their users.

Group ID – This is the Group ID code by which the group is identified within the system.

Group Name – This is the full Group name.

Customer Name – This link allows the user to view the customers within the group.

Delete – Clicking on this icon erases the group from the system.

Edit – This function allows the user to make changes to the customer, users or group details.

Group ID

Clicking on the Group ID field in the Group list allows the user to view the Group Details.

Group ID – This field lists the Group ID code.

Group Name – This field lists the full Group Name.

Group Contact Person – Other Group Details such as the contact person or other pertinent information is listed in this and succeeding fields.

Customers per Group

This page allows the user to view, edit or delete customer and user details. Click on the "Customers Per Group" link in the Group List Form to enter this screen.

The customer section of the page consists of two parts: The customer ID field and the Authorization field.

Customer ID – This field identifies the customer

Authorization – This field indicates what transactions the customer may perform.

Customer Authorization

Transaction – Each transaction that the Bank may allow a customer to perform is listed on this table.

Authorization – The Authorization field indicates whether or not the customer may perform that transaction. The Bank can restrict the authorization of Bank Users causing Segregation of Duties. This will be effective when the Limit Bank User Authorizations field is “Yes” for the Entity in the Customize Entity window. The pages will be different depending on whether the Segregation of Duties exists for the entity.

The user section of the page consists of two parts: The User Id field and the Supervisor field.

User Id - This field identifies the user.

Supervisor - This field indicates whether or not the user is a supervisor and what transactions, as a result, the user may perform.

User Authorization / User Disable Option

This page indicates whether or not the user is a supervisor and provided with the level of authorization that it entails.  Click on the User-id link or the pencil icon in the User area on the User and Customers Details Form  to enter this screen.  NetIMEX will not allow a user that signed transactions to be deleted so his details will remain in the system for the purpose of auditing the signatures, therefore it is also possible to disable a user.  Check the Disable box on the user profile screen to flag the user as disabled.  As a result, disabled users will not appear in the Supervisor's user maintenance function.

Note: Disabling a user in NetIMEX should be performed in parallel to revoking the user's access rights in the bank's security system.

User Details With Segregation of Duties

Click on the "Authorization" link to enter this screen. When Segregation of Duties exists for this entity the authorization of the Bank users are split into two types of Bank Users for the entity:

1.      Supervisor Bank User

A supervisor has the following authorizations:

  • Authorized to create, delete or edit any Supervisor Bank Users or Regular Bank Users. For Regular Bank Users the Supervisor must set the authorization for specific functions e.g. configuration, audit trail access, translation.
  • Can change a Regular Bank User to become a supervisor Bank User by setting the supervisor checkbox. The original Bank User permissions will be deleted.
  • Can only perform specific functions, i.e., all items under Inquiry, Customize Preferences, Languages, Users, Customers, Bank Maintenance – Group only. When a Supervisor Bank User logs in to NetIMEX only these functions will be available.
  • Does not have permission for transactions therefore the signature level for the Bank is “Not Authorized”.
  • The supervisor cannot carry out any of these Bank functions and define any Customer Users.

2.      Regular Bank User

A regular Bank User can work according to the authorization set by the supervisor. Cannot create any Bank Users, not as a supervisor nor as a regular Bank user and cannot give any regular Bank User authorization.

User Details – Defines the User ID and Name

Supervisor – When checked indicates that the user is a supervisor. Even if the permissions are changed to allow they will saved as not allowed.

Disable – Check the Disable box to flag the user as disabled. As a result, disabled users will not appear in the Supervisor's user maintenance function. Disabling a user in NetIMEX should be performed in parallel to revoking the user's access rights in the bank's security system.

Permissions – The following permissions can be given to the Customer Service or Bank User Functions:

  • Allow: Allow to access and update
  • View: View only, allowed to access but not update
  • Not Allow: Not authorized, no access to the function

Only a Regular Bank User can be given permission for these functionalities:

Change Customer – In Edit/Create/Delete Mode the Regular Bank User can do the following for the bank customer user: Transaction inquiry, View signature level, Create/Update templates, View banks, Handle parties, Clauses and Purge. In View Mode the Regular Bank User can do the following for the bank customer user: View Banks, Parties, Clauses, Purge, Transaction inquiry, signature level, templates (cannot create/edit/update templates, will not see any transactions from the menu and in “Customization Templates” the edit and delete icons will be disabled).

User Management – (Group, delete add update view, Group by customer, Customer per group, Delete/Update/View Customer and User). Only in view mode the user will see all the relevant pages. In Edit/Create/Delete Mode Bank users that do not have the permission will not be able to see the group in the bank maintenance page.

Translations – (New Language, Delete Language, Tables Translation, Titles Translation). In view mode it is only relevant for table translation and title translation. In Edit/Create/Delete Mode Bank users that do not have the permission will not be able to see the languages related in the bank maintenance page.

Configuration – (General Parameters, Deal Number for Department, Help Page Configuration, Signature level, Clauses, Customize template). In Edit/Create/Delete Mode Bank users that do not have the permission will not be able to see the configuration pages in the bank maintenance page

Audit Trail – In Edit/Create/Delete Mode Bank users that do not have the permission will not be able to see the audit trail page in the bank maintenance page.

PO Integration – In Edit/Create/Delete Mode Bank users that do not have the permission will not be able to see the 'PO Integration page in the bank maintenance page.

    Tables Translations

    This option allows the Bank to translate NetIMEX tables to the languages provided by the Bank for its customers.

    After clicking on the Tables Translation Option on the Bank’s Maintenance Form, choose a language to which you wish to translate the tables. When the customer chooses to work in this language, only the tables that are translated in this option will be translated and will appear in that language. Click on the name of the table that you wish to translate e.g. Draft After Table. A list of the selected table’s fields appears on the screen.

    Translate each of the fields in this table into the way you want them to appear in that language’s version.

    Click on the next table and continue this process until all the relevant tables have been translated.

    Titles Translation

    This option allows the Bank to translate the menu options to the language chosen by the customer.

    After clicking on the Titles Translation Option on the Bank’s Maintenance Form, choose a language to which you wish to translate the Menu options. When the customer chooses to work in this language, only the menu options that are translated in this option will be translated and will appear in that language. Click on the name of the option that you wish to translate. A list of that option’s headings appears on the screen.

    Translate each of the headings in this option into however you want them to appear in that language’s version.

    Click on the next option and continue this process until all the relevant titles and headings have been translated.

    Release Period For Locking Transaction

    Enables a user to lock a transaction for a defined period of time, between 0 - 9999 minutes.
    The default value of the release period is ‘0’.
    If the release period is ‘0’ only the user that locked the transaction can open it.
    If the release period is, for example, ‘10’, the user has placed a lock on the transaction for 10 minutes, and other users cannot edit the transaction for that period of time. After 10 minutes, the transaction is automatically unlocked, enabling all users to edit the transaction.

    Deal Number for Department

    NetIMEX allows for the automatic generation of the host deal number at the time of transaction initiation.  The deal number is generated prior to sending it to the host and receiving a return acknowledgement.

    The deal number generation is based on the following logic:

    • Each customer is assigned a department ID, per deal type as part of the customer's details, which are downloaded, from the host.
    • For each department NetIMEX maintains a per deal type numerator.
    • At transaction initiation the relevant department per deal type numerator is incremented accordingly.

    This function is used to setup the initial value of the numerator. Click on the Deal Number for Department link.  The list of departments is displayed. The list includes the department numbers and names.

    Updating the Department Numbers

    From the department click on the Edit icon  .

    The Deal Number for Department maintenance page is displayed.

    The Deal Number for Department maintenance page displays a number of fields which show the current numerator for all transactions that can be initiated in NetIMEX: Import Letter of Credit, Guarantees, Remittance, Documentary Collections, Clean Collections.

    Enter the new value for the required deal type(s) and click on the Save button 

    Note: To specify that no deal number should be generated for a deal type, leave the deal type number field empty.

    Viewing the Department Numerators

    From the department list, click on the department name link.  The Deal Number for Department page is displayed.

     

    Help Page Configuration

    NetIMEX can specify a Help page or Help application for each NetIMEX page different than the Help page delivered with Net IMEX making it unnecessary to override the supplied Help pages with the Bank’s Help pages.

    The special Help Path item allows the Bank to specify where the Help page is located is so that the NetIMEX Help installation location can remain intact. In the Help Path specification a special token <Lang> defines if the new location is language sensitive (having a specific directory for each NetIMEX language defined) or it is a only one Language (in this case the <Lang> sub-path does not need to be specified in the Help Path specification).

    An additional parameter in the General configuration page allows the Bank to specify whether Help pages are displayed in a separate window.

    Audit Trail

    The Bank can configure what type of activities are to be logged and can generate an inquiry on the audited information.

    A manual purge facility allows the bank to delete audit information from the database using flexible criteria. There is also a background service for automatic purge based on a retention period.

    The following activities can be logged:

    • Transaction Processing
    • User Profile Maintenance
    • User authorization changes
    • Signature Levels Changes
    • Banks and Parties Maintenance
    • Bank General Parameters Maintenance
    • Translation

    Audit Trail Inquiry

    The Bank can inquire upon the audited information using a flexible search facility for the following criteria: Activity, Customer, User and/or Dates From and To. In order to display a list of all the audited information the user must leave all the criteria fields empty and click on the search button.

    Audit Trail List

    The user can sort the list per column by clicking on the column heading.

    Configure Log Activities

    In order to specify the activities to be logged:

    • Click on the Customize Icon   on the previous screen to view all the items.
    • Check the relevant activities. To check all the items check the Select All checkbox.
    • Click on the Save button to save the changes.
    • Click on the Cancel button to return to the previous screen without saving any changes.

    Clear Log Activities

    The user may clear the Log Activities for an Activity, Customer and/or User for a specific period.

    In order to clear Log Activities:

    • Click on the Delete Icon   on the previous screen.
    • Select the relevant criteria.
    • Click on the Delete button.

     

    PO Integration

    The integration of NetIMEX with PO systems is specifically tailored for each Bank.

    One of the integration functions is the ability to send details of NetIMEX templates to the PO system so it can generate L/C applications based on a specific template. The forwarding of the template details to the PO system is a background ongoing process.

    This function allows the retransmission of the template details to the PO system.

    In order to send NetIMEX templates to the PO system:

    • Click on the PO Integration link on the Bank Maintenance page to display the Bulk Templates Transfer Request Window.
    • Select a Customer or ‘All’ for the whole list of customers.
    • Click on the Send button.

    Clauses

    The Bank can create standard clauses or text library, which can be used by the customer to facilitate the data entry in the Text Windows on the Transaction.

    The Bank creates the text of clauses and defines to which of the text windows and transactions each clause is relevant.

    In the Text section of the transaction, the Open Clause Library icon  indicates that there are clauses relevant to the window. When clicking on this icon a list of defined clauses relevant for the window are displayed. After selecting the required clauses the contents of the clause is copied to the window.

    Select Clause from the Customize section of the Inbox to display the Customize Clauses Search Screen.

    The user has the following options:

    • Create a new clause by clicking on the Add button.
    • Search for a specific Clause by entering the Identification Code of the relevant Clause in the Clause ID field and clicking on the Search button.
    • Display a list of all existing clauses by leaving the Clause ID field empty and clicking on the Search button.

    On the list of existing clauses the user can carry out the following options:

    Edit an existing clause by clicking on the Edit icon. 

    Delete an existing clause by clicking on the Delete  icon.             

    How to add a clause:

    1.       Click on the Add button on the Customize Clause Criteria screen to display the Clause Details Screen.

    2.       Enter the Clause ID.

    3.       Enter the Clause Description.

    4.       Enter the Clause Text. If the text of the clause is more than 4 lines click on the Zoom Text Icon to view/edit all the data.

    5.       In the Clause Availability section check the relevant checkboxes. This will indicate on which window and in which transaction the clause will be available.

    6.       Click on the Save button.

    Defined Clause Descriptions

    An additional option exists to display clauses as defined in the clauses form, which the user can select by selecting the desired checkboxes. This replaces the use of the Clause Library and free text windows.

    There are two options for this function, which is established during bank setup:

    • The user can select more than one clause from one text window.
    • The user can select only one clause from one text window.

    In a text window which is marked as mandatory, a validity check is performed to ensure that at least one clause is selected.

    Purge

    The Purge function in NetIMEX allows the Bank to purge old data. To access the Purge operation, click on the Purge option from the Customize section of the main menu.

    Purge is allowed for the following items:

    • Closed Deals
    • Rejected Deals
    • Incoming Correspondence Items
    • Unsent Transactions

    For each object type, you are presented with the following selection criteria request. 

    • Deal Type - Identifies the type of deal.  Leave this field blank to select all the deal types.
    • Range Dates - Range of expiry dates (for relevant deal types).
    • Search all Customers - When checked the items of all the customers are displayed.

    After selecting the object type and criteria, click on the Search button to display; the list of items matching the criteria will be displayed.

    Note: Items are purged physically. You can restore the data by re-transmitting the relevant deal steps from the back office system.

    Purge - Rejected Deals

    Select the items to purge by clicking the check box to the left of the entry (or select all the items), and request the purge function to be executed by clicking on the Purge button.

    Deal No. - Identifies the Deal number.

    Reference No - A unique deal identification number. 

    ISS/EFF Date - The issuance or effective date of the deal.

    Mat/Exp Date - Specifies the maturity or expiry date of the deal.

     

    Purges the selected entries in the list.

     

    Returns to the previous screen without making any changes.
