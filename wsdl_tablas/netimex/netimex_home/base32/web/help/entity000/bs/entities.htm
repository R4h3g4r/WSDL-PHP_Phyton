ENTITIES

Introduction

The NetIMEX Multi-Entity feature is designed to allow a Bank using NetIMEX to host the electronic Trade Finance operation of several Entities in one installation of the system, rather than having several different installations - one for each entity.

The appearance for each Entity created is copied from an existing Entity.  The User creating the entities can select the Entity from which to copy the appearance of the screens in the new Entity. All the necessary tables, images, help files and other information required by the system is automatically copied to the new Entity on its creation.

NetIMEX is supplied with one General Bank User defined. This user can perform only two actions in the system:

  • Maintain General Bank Users. These users will be able to perform these same two functions only.
  • Create new Entities.

A customer User will automatically be logged in to the Entity to which he belongs, and will be able to view and operate only those Customers belonging to his group, according to the authorization given to him. The regular Bank user has the option to edit, create and customize certain features for his Entity. For further details please refer to the section on Bank User Work Options with Entities.

Customizing Entities

The first screen to which the General Bank User is routed after login is the Customize Entity screen, where a list of all Entities is displayed and Entities can be created, edited or deleted.

Entity ID - This is the Entity ID code by which the Entity is recognized within the system.

Entity Description - This is the full name of the Entity.

Limit Bank User Authorization - Allows the Bank to restrict the authorization of Bank users.

  • If Yes, segregation of duties exists for this entity. This means that the authorization of the Banks users are split into two types of Bank Users for the entity:

- Supervisors: Authorized to create other Bank Users as a supervisor or regular Bank user. For regular user the Supervisor must set the authorization for specific functions e.g. authorization maintenance, configuration, audit trail access, translation. The supervisor cannot carry out any of these Bank functions and define any end Users (None Bank User). 

- Bank User. Can work according to the authorization set by the supervisor. Cannot create any Bank Users, not as a supervisor nor as a regular Bank user and cannot give any regular Bank User authorization.

  • If No, segregation of duties does not exist for this entity.

Delete  - Clicking on this icon will erase the Entity from the system.

Note: An Entity can only be deleted from the system if there are no regular customers for tlis Entity defined in the system.

Edit - This function allows the user to make changes to the Entity Description.

 Clicking on this button allows the user to create a new Entity,

 Returns the user to the previous screen without making any changes.

General Bank Users List - Click on this link to display the list of all General Bank Users in the system.

Add Entity

To add a new Entity, click on the  button at the bottom of the Entities list. The Entity Details form is displayed.

To complete the Entity Details form, enter the required data in the relevant fields. All the fields in this form are mandatory.

Entity ID - Enter a 3 character Entity ID by which the Entity will be recognized in the system.

Entity Name - Enter the full name of the Entity.

Copy Layout from Entity - Select the Entity from which the Layout is to be copied. The layout can be copied from any Entity that already exists in the system.

Limit Bank User Authorization - Allows the Bank to restrict the authorization of Bank users.

  • If Yes, segregation of duties exists for this entity. This means that the authorization of the Banks users are split into two types of Bank Users for the entity:

- Supervisors: Authorized to create other Bank Users as a supervisor or regular Bank user. For regular user the Supervisor must set the authorization for specific functions e.g. authorization maintenance, configuration, audit trail access, translation. The supervisor cannot carry out any of these Bank functions and define any end Users (None Bank User). 

- Bank User. Can work according to the authorization set by the supervisor. Cannot create any Bank Users, not as a supervisor nor as a regular Bank user and cannot give any regular Bank User authorization.

  • If No, segregation of duties does not exist for this entity.

 Click to save the new Entity in the System. This new Entity will automatically be assigned the number 000.

 Click to return to the Entities list without saving any changes.

Note: When a new Entity is created, the system automatically creates a General Bank User called SYSTEMXXX where XXX is the ID of the newly created Entity, using the default Bank User password. Using this ID and password, the new General Bank User can log on to the system and perform those functions for which he is authorized

Edit Entity

Click on the Edit icon to edit the Entity details. The Entity Details form is displayed.

The only field that can be edited on this form is the Entity Name. Enter a new name or change the existing name, as required.

 Click to save the changes to the Entity Name and return to the previous screen.

 Click to return to the Entities List without saving any changes.

General Bank User List

Click the General Bank Users link at the bottom of the Entities list in order to display the list of the General Bank Users in the system. The general bank User can perform only two actions in the system:

  • Create other General Bank Users.
  • Create Entities.

Note: There must be at least one General Bank User per Entity and the user will not be able to delete a General Bank User if he/she is the only one.

User ID - The User ID code by which the User is recognized in the System.

Delete  - Deletes the General User from the system, provided that he/she is not the only General User for that Entity.

Update - Enables the User to edit the General User Details.

 Click to add a General Bank User to the system.

 Click to return to the previous screen without saving any changes.

Click on the Entities link at the bottom of the General Bank Users List to return to the Entities List.

Regular Bank User Work Options with Entities

Customer

  • The regular Bank user can view and work with only those Customers defined in his Entity.
  • The regular Bank user can create new Customers and edit existing Customers only for his own Entity. The Entity for the newly created Customer will be set according to the Entity of the Bank User who created the Customer. The system will identify this Entity from the information entered at login.

Group

  • The regular Bank user can view and work with only those Groups defined in his Entity.
  • The regular Bank user can create new Groups and edit existing Groups only for his own entity. The Entity for the newly created Group will be set according to the Entity of the Bank user who created the Group. The system will identify this Entity from the information entered at login.

User

  • The regular Bank user can view and work with only those Users defined in his Entity.
  • The regular Bank user can create new Users and edit existing Users only for his own entity. The Entity for the newly created User will be set according to the Entity of the Bank user who created the User. The system will identify this Entity from the information entered at login.

Customization

The regular Bank user can customize certain features, for his Entity only. For details on customizing features, please refer to the section on Bank Maintenance.
