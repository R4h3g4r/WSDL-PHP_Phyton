Open Deals

The Open Deal Inquiry option allows the user to view and edit any deal which has a balance (such as deal balance, payment balance, acceptance balance).  To view the open deals window, select the Open Deals option from the Inquiry menu or scroll down the Inquiry page.

Open deal inquiries are made according to the following parameters:

Open Types – This field is mandatory for Deal selection. These are the open types of transactions in the system, which have not been completed. They may include: Acceptances, Collections, Payments, Guarantees, L/C, Loans, Margins Payments.

Deal Type – Deal Types include the kind of deal and the step: They may include Export Clean Collections, Export Clean Payments, Export Credit Bills, Export Documentary Collections, Export L/Cs, Import Clean Collections, Import Clean Payments, Import Documentary Collections, Import L/Cs, Outgoing Guarantees and Shipping Guarantees.

Currency – The currency in which the deal amount is expressed. They are the options available to the customer as determined by the Host Bank.

Amount Range – The lowest and highest credit limits for deals as set by the Bank.

Local Currency – Allows the user to select the Local Currency to be used to calculate the “Amount in Local Currency” fields in the open deals result table. The rates for the conversion are taken from the Rates Table.

If the Local Currency combo box is left empty, the local currency equivalent calculations will use the information contained in the Customer Local Currency field, if defined, from the Customize Customer screen.

    Expiry Date Range – From/To (relevant to an open L/C). These are the dates during which the rate provided by the Bank remains in force.

    Maturity Date Range – From/To (relevant to an open acceptance and finance).

Counterparty – An ISO term. The corresponding party to the transaction. The Party with whom a contract has been concluded, i.e. client, broker, other branch, H.O., other bank or a department.

All Customers - Check this box if the request is for a groups of Customers or only for the Customer the user is currently working with.

Note: The availability of this field is dependent upon the user having permission to activate inquiries at the group level.

Save Criteria - Check this box for NetIMEX to save the inquiry criteria as the default display in the user profile All subsequent requests for the inquiry will use the saved criteria as the default.

If the user wants to remove a saved criteria status for the next default Transaction Status, thereby creating an empty criterion status, the following step needs to be taken:

  • All fields need to be blank, except the Save Criteria field, which need to be checked, and then click Search.
A user is allowed to use one criterion per inquiry.
Click on this button to begin the search. A list of transactions that conform to the retrieval criteria appear in the Inquiry Results Table as shown in the next screen.
When the user clicks on this button the list of inquiry results are exported to Excel, which will automatically be invoked from the user’s browser. The user can then manipulate the downloaded data, including saving it.

Inquiry Open Deals Results Table

The top of the report lists the search criteria that defined the report parameters. The bottom of the page shows the search results.

 

 

After receiving the inquiry results, the user may sort them by any of the Inquiry Results table column options. To sort the Inquiry Results Table:

  • Click on the   button for ascending sort order; or
  • Click on the   button for descending sort order
  • Then select any column with a blue heading.

To view a transaction, double click on a transaction record in the Transaction Name column.

To view a list of open deals that conform to a user defined retrieve criteria, specify the relevant parameters and click the Search button.

Deal No – A unique Reference Number specified by the Bank at issuance

Drw No – Drawing Number. The Drawing Number is the same as the Transaction Number. It indicates how many actions have been performed on the deal application process.

ISS/EFF Date – The Issue Date is the date on which the deal goes into force.

Balance – The remaining amount of the deal to be paid.

Amount in Local Currency – The original amount of the deal converted to local currency.

Balance in Local Currency – The amount of the deal balance converted to local currency

Customize Output Layout

When running an Open Deals Inquiry, it is possible to customize the output so as to view only those columns that you wish to, and not all the columns according to the default parameters.

To Customize the Output

1.       Click the Select Output Layout icon   on the Open Deals Inquiry Screen. The Personalize Page Layout Dialog box is displayed.

 

2.       From the list of default columns, select the column you wish to appear in your customized sort. Click the right arrow to move your selection to the selected columns, which appear in the right hand pane of the window.

3.       Click the Select icon   to save your choice and return to the main Open Deals Inquiry window. Click   to return to the Open Deals Inquiry without making any changes.

4.       The results of the next Open Deals Inquiry you run will be displayed according to your customized layout.

Note: If no selection is made for customization, the display will be according to the default NetIMEX standard.
