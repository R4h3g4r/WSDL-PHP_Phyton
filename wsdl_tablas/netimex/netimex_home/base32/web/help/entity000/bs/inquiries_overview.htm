INQUIRIES

Overview of Inquiries

This chapter includes details about making deal inquiries according to user-defined criteria. The following types of inquiries are explained in this chapter:

  • Transaction Status Inquiry - Inquiries about transaction status.
  • Open Deals Inquiry - Inquiries regarding transactions which are in process.
  • Deal History Inquiry - Inquiries relating to a specific transaction, including the original data and all transmissions relating to that transaction.
  • Deal Select Inquiry - General deal inquiries which are about selected groups of deals.
  • Inquiry Request - Report request inquires.

The Inquiries are listed in the Main Menu, which appears on the left side of the page. The Inquiry Request Form is displayed on the right side of the page as shown in the following screen.

To submit an Inquiry, click on the Inquiry link on Menu.

Transaction Status Inquiry

The first Inquiry option is the Transaction Status Inquiry. This option enables the user to view a list of customer initiated transactions  The Transaction tracks:

  • Viewing details pertaining to the signatures required for the current transaction.
  • Viewing a list of the signatures that have signed a transaction.
  • Tracking changes in the transactions.

To view the Transaction Status Request Form, select the Transaction Status option from the Inquiry menu or enter details in the Transaction Status request form that appears when you open the Inquiry section.

Status inquiries are requested according to the following parameters:

  • Transaction: This section of the Inquiry describes factors relating to the transaction itself.
  • Status: Since each change affects a transaction’s status, this report lists when the transaction changed to the current status.

The transaction section requires that two fields are entered:

Status – This field limits the search to the current transaction stage. Thus, transactions may be identified by what actions were performed on them or in light of what stage they were assigned, for example: In process, Released, Signature Required etc.

Name – This field identifies the transaction deal type and step.

The Status section requires that a further two fields are entered:

Date from/to - The date parameter restricts the search to a range of when the transaction changed to the current status.

All Customers - Check this box if the request is for a group of Customers or only for the Customer the user is currently working with.

Note: Availability of this field is dependent on the user having permission to activate inquiries at the group level.

Click on this button to begin the search. A list of transactions that conform to the retrieval criteria appear in the Inquiry Results Table as shown in the next screen.
When the user clicks on this button the list of inquiry results are exported to Excel, which will automatically be invoked from the user’s browser. The user can then manipulate the downloaded data, including saving it.

Open Deals

The Open Deal Inquiry option allows the user to view and edit any deal which has a balance (such as deal balance, payment balance, acceptance balance).  To view the open deals window, select the Open Deals option from the Inquiry menu or scroll down the Inquiry page.

Open deal inquiries are made according to the following parameters:

Open Types – This field is mandatory for Deal selection. These are the open types of transactions in the system, which have not been completed. They may include: Acceptances, Collections, Payments, Guarantees, L/C, Loans, Margins Payments.

Deal Type – Deal Types include the kind of deal and the step: They may include Export Clean Collections, Export Clean Payments, Export Credit Bills, Export Documentary Collections, Export L/Cs, Import Clean Collections, Import Clean Payments, Import Documentary Collections, Import L/Cs, Outgoing Guarantees and Shipping Guarantees.

Currency – The currency in which the deal amount is expressed. They are the options available to the customer as determined by the Host Bank.

Amount Range – The lowest and highest credit limits for deals as set by the Bank.

Local Currency – Allows the user to select the Local Currency to be used to calculate the “Amount in Local Currency” fields in the open deals result table. The rates for the conversion are taken from the Rates Table.

    Expiry Date Range – From/To (relevant to an open L/C). These are the dates during which the rate provided by the Bank remains in force.

    Maturity Date Range – From/To (relevant to an open acceptance and finance).

Counterparty – An ISO term. The corresponding party to the transaction. The Party with whom a contract has been concluded, i.e. client, broker, other branch, H.O., other bank or a department.

All Customers - Check this box if the request is for a groups of Customers or only for the Customer the user is currently working with.

Note: The availability of this field is dependent upon the user having permission to activate inquiries at the group level.

Click on this button to begin the search. A list of transactions that conform to the retrieval criteria appear in the Inquiry Results Table as shown in the next screen.
When the user clicks on this button the list of inquiry results are exported to Excel, which will automatically be invoked from the user’s browser. The user can then manipulate the downloaded data, including saving it.

Customize Output Layout

When running an Open Deals Inquiry, it is possible to customize the output so as to view only those columns that you wish to, and not all the columns according to the default parameters.

To Customize the Output

1.       Click the Select Output Layout icon   on the Open Deals Inquiry Screen. The Personalize Page Layout Dialog box is displayed.

 

2.       From the list of default columns, select the column you wish to appear in your customized sort. Click the right arrow to move your selection to the selected columns, which appear in the right hand pane of the window.

3.       Click the Select icon   to save your choice and return to the main Open Deals Inquiry window. Click   to return to the Open Deals Inquiry without making any changes.

4.       The results of the next Open Deals Inquiry you run will be displayed according to your customized layout.

Note: If no selection is made for customization, the display will be according to the default NetIMEX standard.

 

Deal History

To View Deal Details

The Deal History Inquiry enables you to view a deal in detail. To view a deal, select the Deal History option from the Inquiry menu. The Deal History Request Form appears.

 

Enter the deal number and/or your reference number (and deal type) as appropriate. Click the Search button to view the deal.

 

The Deal Details appear together with a history of transactions performed on that deal. The deal history options enable you to view details entered by the customer and bank’s host.  This option is only active if you originated the transaction.  The transaction can be either an import or export transaction.

The following information may appear together with the deal details, depending on the nature of the deal:

  • Original Transaction Details (initiated by the customer)
  • Deal Participants
  • Settlement Data
  • Deal Texts
  • Deal Open Payment Details
  • Deal Open Acceptance Details
  • Deal Open Finances
  • Deal Open Margins

Note: The information that is available in the search results depends on the nature of the particular transaction.

Deal Select

The Deal Selection inquiry displays all deals that conform to the select parameters.

To view the deal selection window, choose the Deal Select option from the Inquiry menu or scroll down the Inquiry page. 

Deal selection inquiries are made according to the following criteria:

  • Deal Number
  • Reference or Invoice Number
  • Deal Type
  • Counterparty
  • Currency
  • Amount Range

Check the All Customers checkbox in order to activate the inquiry for a group of Customers. The availability of this field is dependent on the user having permission to activate inquiries at the group level.

To view a list of deals, specify the relevant parameters and click the Search icon.

To perform a general search to view all the deals sent to or received from the bank, click on the Search button without entering any of the fields.

 

To view a deal in greater detail, click on the individual Transaction in the Deal Type column.  The results of the search may be sorted by any of the heading topics found in the Table Heading. Double click on that column heading to perform a sort.

Inquiry Request

This feature will enable customers to make a request for a back office generated report.  A back office generated report in this context means a back office document output which is not part of a deal step release, such as the Customer Limit Exposure report.  Click on the Inquiry Request option from the Inquiry section of the main menu. 

 

Note: When there are no entries available to be shown, the Inquiry Request option will not appear in the menu.

The Inquiry Request form will appear.

The available reports will be listed.  Click on the desired report and press Send.  The report request is sent to the Back Office and the result is viewed in the Incoming Correspondence section of the Inbox.
