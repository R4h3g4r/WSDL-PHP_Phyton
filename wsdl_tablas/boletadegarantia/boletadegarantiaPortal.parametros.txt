Esta tabla contiene especificamente los parametros utilizados en Boletas de Garantia Portal.

ROR
~~~
- En la línea +tipoCliente+ se definen valores codigo de los tipos de cliente beneficiarios de una boleta que,
podrían o no estar registrados en la tabla ROR. Si se trata de una persona natural el codigo de +persona+
tendra el valor "+P+", si se trata de una +empresa+ el codigo sera el valor "+E+" y finalmente si no esta
registrado se utilizara la llave +noExiste+ con el codigo +N+.

- En la línea +tipoRelacion+ se definen valores codigo de los tipos de relacion que participan en una boleta
los cuales serán campos claves de busqueda en la tabla ROR, de tal manera que cuando se haga la consulta,
u otra transaccion sera inpresindible usar un tipo de relacion.Se definen de manera de parametrizar los
valores utilizados.

Dependiendo del tipo de relacion ingresado se hara la consulta; si la variable de +tipoRelacion+ tiene:
El valor "+000+" se refiere a un beneficiario.
El valor "+001+" se refiere a un retirante de la boleta en la primera vez.
El valor "+002+" se refiere a un retirante de la boleta que reimprime n veces una boleta.

Ejemplo de uso del tipo de relacion:
Se define el tipo de relacion que se requiere:
---------------------------------------------

tipoRelacion = TablaValores.getValor(archivo_parametros, "beneficiario", "tipoRelacion");
---------------------------------------------
Se realiza la consulta.
---------------------------------------------
resultBNF = creditosglobales.consultaBeneficiario(multiEnvironment,
iicNumero,
Integer.parseInt(boletaNumero),
tipoRelacion,
Integer.parseInt("0"),
' ',
' ',
" ");
---------------------------------------------

- La línea +singlosa+ se define un +valor+ que estara por defecto en caso de que no se posea glosa que en este
caso será "+sin glosa+".

- La línea +sindireccion+ se define un +valor+ que estara por defecto en caso de que no se posea direccion que
en este caso será "+sin direccion+".

- En la línea +etapasBex+ se definen valores codigo de las etapas de una boleta los cuales serán los necesarios
para ingresos y busqueda de boletas.Se definen de manera de parametrizar los valores utilizados.

Dependiendo de la etapa de la boleta se tiene que:
El valor "+PEM+" se refiere a la etapa de la boleta cuando esta pendiente por emitir.
El valor "+EMT+" se refiere a la etapa de la boleta emitida.
El valor "+EMR+" se refiere a la etapa de la boleta emitida y reimpresa.

- En la línea +estadosBex+ se define el estado de una boleta vigente tendrá como valor codigo "+VIG+".

- En la línea +estadosOpe+ se define el estado de una operacion activada aprobada tendrá el valor codigo "+ACA+".

- La linea +emisor+ contiene el Valor y el tipo de relacion definido para grabar el registro en entidad ROR.
Dependiendo del tipo de cliente se utilizara el valor asociado tal como se muestra en la cita del codigo siguiente:
---------------------------------------------
if (tipoCliente.equalsIgnoreCase(TablaValores.getValor(archivo_parametros, "beneficiario", "valor"))){
tipoRelacion = TablaValores.getValor(archivo_parametros, "beneficiario", "tipoRelacion");
}
else if (tipoCliente.equalsIgnoreCase(TablaValores.getValor(archivo_parametros, "emisor", "valor"))){
tipoRelacion = TablaValores.getValor(archivo_parametros, "emisor", "tipoRelacion");
}
---------------------------------------------
El valor "+EMI+" es para emisor.
El valor "+BNF+" es para beneficiario.

- En la línea +estadosProcFirma+ se definen los codigos de los estados de las firma que podrian tener en una
boleta los cuales son utilizados entre otros para hacer el Log de la firma.
El valor "+FDF+" se refiere a una boleta que no ha pasado aun por el proceso de la Firma.
El valor "+FIR+" se refiere a una boleta que fue firmada satisfactoriamente.
El valor "+FAC+" se refiere a una boleta que fue firmada satisfactoriamente y ademas Activada.
El valor "+FAD+" se refiere a una boleta que fue firmada satisfactoriamente, Activada y registrada en la Tabla DLC.

Impresion
~~~~~~~~~
Se definen dos lineas con los parametros necesarios en las plantillas de impresion para pesos y UFs respectivamente,
la línea +pesos_plantilla+ conteniendo los valores de los parametros para crear el PDF de Boletas en Pesos y la línea
+uf_plantilla+ conteniendo los valores de los parametros para crear PDF de Boletas en UFs.
Se definen las dos plantillas pues dependiendo de la unidad de moneda tendra variaciones de mas o menos campos.

La llave +ruta+ define la ruta que contendra la plantilla.
La llave +plantilla+ define el nombre de la plantilla.
La llave +tagPDF+ define el tag utilizado dentro de la plantilla para su definicion.

Se definen dos lineas para las imagenes de las firmas utilizadas en las plantilla para visualizarse en el PDF
generados desde estas plantillas.

La llave +ruta+ define la ruta que contendra la imagen jpg.
La llave +imagen+ define la ruta que contendra el nombre de la imagen jpg.

Mensajes
~~~~~~~~
Esta sección define mensajes de retorno por la consulta de alguna consulta no existente, la linea que los identifica
es +mensajeError+ de pendiendo del caso la llave +noPesos+ para la boleta no habilitada para la unidad de moneda
solicitada y la llave +noEmisor+ para un emisor inexistente.

Reimpresion
~~~~~~~~~~~
Para el proceso especifico de la reimpresion n veces es necesario identificar varios flag para el proceso, y se
paramtrizan a fin de no utilizar valores en duro.
La linea +reimpresion+ posee la llave +reemitida+ que se codifica usando el valor "+R+" y para la llave +flag+
el valor "+1+" indica encendido.
